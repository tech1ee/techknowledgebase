# Research Report: KMP CI/CD

**Date:** 2026-01-03
**Sources Evaluated:** 15+
**Research Depth:** Deep

## Executive Summary

GitHub Actions — основной CI/CD для KMP. iOS билды требуют macOS runner ($0.08/min vs $0.008/min Ubuntu). Кэширование ~/.konan критично для скорости. Fastlane упрощает iOS signing и App Store Connect upload. Отдельные jobs для Android и iOS с dependency chain для cost optimization. KMMBridge от Touchlab для XCFramework distribution через SPM. Bitrise предлагает KMP-оптимизированные workflows.

## Key Findings

1. **GitHub Actions Structure**
   - Separate jobs for JVM/Android (ubuntu) and iOS (macos)
   - iOS costs 10x more ($0.08 vs $0.008 per minute)
   - Run iOS tests only after JVM tests pass (needs:)
   - Tag-based release triggers (v*-ios, v*-android)

2. **Caching Strategies**
   - ~/.konan — Kotlin/Native tooling (~1GB)
   - ~/.gradle/caches — Gradle dependencies
   - Cache key: hashFiles('gradle/libs.versions.toml')
   - gradle/actions/setup-gradle handles Gradle cache

3. **iOS Code Signing**
   - apple-actions/import-codesign-certs for certificates
   - apple-actions/download-provisioning-profiles for profiles
   - App Store Connect API keys stored in secrets
   - Fastlane match for team certificate management

4. **Android Signing**
   - Keystore as base64 in secrets
   - Decode and pass to Gradle
   - r0adkll/upload-google-play for Play Store

5. **KMMBridge**
   - XCFramework publishing to GitHub Releases, Maven, S3
   - SPM Package.swift generation
   - iOS developers consume via Swift Package Manager

6. **Bitrise**
   - Native KMP support with optimized workflows
   - Automatic pipeline generation for new projects
   - Built-in caching and artifact management

## Community Sentiment

### Positive
- GitHub Actions well-suited for KMP
- Fastlane simplifies iOS complexity
- KMMBridge solves XCFramework distribution
- Caching significantly speeds up builds

### Negative
- macOS runners expensive (10x Linux)
- iOS signing complexity
- Kotlin/Native builds still slow
- First builds without cache are very long
