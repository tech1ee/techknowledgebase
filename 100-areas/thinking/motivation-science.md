# Наука мотивации: почему награды не работают

Премия за креативную работу снижает креативность. Оплата за хобби превращает его в обязанность. Внешние награды могут уничтожить внутреннее желание. 40+ лет исследований Self-Determination Theory объясняют почему: устойчивая мотивация требует автономии, компетентности и связи с другими. Без них любая система бонусов работает временно и создаёт зависимость от внешних стимулов.

---

## TL;DR

```
КЛЮЧЕВЫЕ ИДЕИ:

1. Внутренняя мотивация > внешней (по всем параметрам)
2. Три потребности: автономия + компетентность + связь с другими
3. Награды за интересную деятельность СНИЖАЮТ мотивацию
4. Внешнюю мотивацию можно интернализировать
5. Мотивация — ресурс, который истощается
```

## Теория самодетерминации (Self-Determination Theory)

### Что это

Self-Determination Theory (SDT) — наиболее исследованная и валидированная теория человеческой мотивации. Разработана Эдвардом Деси и Ричардом Райаном с 1980-х годов.

> **40+ лет исследований**, тысячи публикаций, применение в образовании, бизнесе, спорте, здравоохранении, гейм-дизайне.

### Ключевое открытие

Исследования SDT показывают:

```
Внутренняя мотивация предсказывает:
─────────────────────────────────────
• Более глубокое обучение
• Более высокую креативность
• Лучшую производительность (особенно в сложных задачах)
• Большее психологическое благополучие
• Более устойчивое поведение во времени

Внешняя мотивация предсказывает:
─────────────────────────────────────
• Быстрое выполнение простых задач
• Зависимость от постоянных наград
• Снижение качества при усложнении задачи
• Прекращение деятельности при отмене награды
```

### Почему это важно для разработчиков

```
Типичная ситуация:

Компания: "Внедрим бонусы за закрытые тикеты!"
         ↓
Результат: Много закрытых тикетов
         ↓
Качество: Падает
         ↓
Творческие решения: Исчезают
         ↓
Техдолг: Растёт
         ↓
Разработчики: "Работаю ради бонуса, не ради продукта"
```

## Внутренняя vs внешняя мотивация

### Определения

**Внутренняя (intrinsic) мотивация:**
Деятельность выполняется ради неё самой — потому что она интересна, приятна, удовлетворяет любопытство.

```
Примеры внутренней мотивации:
• Программист ночью ковыряется с новым языком — просто интересно
• Геймер проводит часы в игре — это увлекательно
• Музыкант играет для себя — это приносит удовольствие
• Open source контрибьютор фиксит баги — хочется сделать лучше
```

**Внешняя (extrinsic) мотивация:**
Деятельность выполняется ради внешнего результата — награды, избегания наказания, одобрения.

```
Примеры внешней мотивации:
• Работа ради зарплаты
• Учёба ради диплома
• Спорт ради медали
• Код ради премии
```

### Континуум мотивации (не бинарность!)

SDT показывает: мотивация — это не "внутренняя или внешняя", а спектр:

```
КОНТИНУУМ МОТИВАЦИИ (Ryan & Deci, 2000)
══════════════════════════════════════════════════════════════════════════

МЕНЬШЕ                    САМОДЕТЕРМИНАЦИИ                         БОЛЬШЕ
◄─────────────────────────────────────────────────────────────────────────►

┌─────────────┬─────────────┬─────────────┬─────────────┬─────────────────┐
│  АМОТИВАЦИЯ │  ВНЕШНЯЯ    │ ИНТРОЕЦИР.  │ ИДЕНТИФИЦ.  │  ИНТЕГРИРОВ.    │
│             │  РЕГУЛЯЦИЯ  │  РЕГУЛЯЦИЯ  │  РЕГУЛЯЦИЯ  │  / ВНУТРЕННЯЯ   │
├─────────────┼─────────────┼─────────────┼─────────────┼─────────────────┤
│   "Зачем?"  │   "Надо,    │   "Должен,  │  "Это важно │  "Хочу, это     │
│   "Бессмыс- │    иначе    │    иначе    │   для моих  │   часть меня"   │
│    ленно"   │   накажут"  │   стыдно"   │   целей"    │                 │
├─────────────┼─────────────┼─────────────┼─────────────┼─────────────────┤
│  Нет        │  Полностью  │  Частично   │  Осознанно  │  Полностью      │
│  мотивации  │  внешняя    │  внешняя    │  принятая   │  внутренняя     │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────────┘

Пример для разработчика:

АМОТИВАЦИЯ:     "Не понимаю, зачем я это делаю. Бессмысленная задача."
ВНЕШНЯЯ:        "Делаю, потому что платят / уволят."
ИНТРОЕЦИР.:     "Делаю, потому что стыдно не сделать / все ожидают."
ИДЕНТИФИЦ.:     "Делаю, потому что понимаю ценность для проекта."
ИНТЕГРИРОВ.:    "Делаю, потому что это часть моей профессиональной идентичности."
ВНУТРЕННЯЯ:     "Делаю, потому что это интересно и увлекательно."
```

### Практическая разница

```
Задача: Рефакторинг легаси-кода

Разработчик A (внешняя):          Разработчик B (внутренняя):
─────────────────────────         ────────────────────────────
"Сколько заплатят?"               "Интересно разобраться"
Делает минимум                    Копает глубже
Заканчивает при первой            Находит неочевидные проблемы
  возможности                     Получает удовольствие
Чувствует облегчение              Хочет продолжить

Результат A: Формально сделано    Результат B: Реальное улучшение
```

## Три базовые психологические потребности

### Главное открытие SDT

Внутренняя мотивация расцветает, когда удовлетворены три базовые потребности:

```
                    ╔════════════════════════════════════════╗
                    ║    ТРИ БАЗОВЫЕ ПОТРЕБНОСТИ (SDT)       ║
                    ╚════════════════════════════════════════╝

           ┌─────────────────┐
           │    АВТОНОМИЯ    │
           │                 │
           │  Чувство выбора │
           │  и контроля     │
           └────────┬────────┘
                    │
                    │    Когда все три
                    │    удовлетворены
                    ▼         ↓
    ┌───────────────────────────────────────────┐
    │                                           │
    │         ВНУТРЕННЯЯ МОТИВАЦИЯ              │
    │              расцветает                   │
    │                                           │
    └───────────────────────────────────────────┘
                    ▲
                    │
        ┌───────────┴───────────┐
        │                       │
┌───────┴───────┐       ┌───────┴───────┐
│ КОМПЕТЕНТНОСТЬ│       │   СВЯЗНОСТЬ   │
│               │       │ (Relatedness) │
│ Чувство       │       │               │
│ эффективности │       │ Чувство       │
│ и мастерства  │       │ принадлежности│
└───────────────┘       └───────────────┘
```

### 1. Автономия (Autonomy)

**Что это:** Ощущение, что ты сам выбираешь свои действия, а не являешься марионеткой внешних сил.

```
Автономия ≠ Независимость

Автономия — это ощущение "я выбрал это сам"
Можно работать в команде и чувствовать автономию
Можно работать один и НЕ чувствовать автономию (если задача навязана)
```

**Что поддерживает автономию:**
```
✓ Выбор в том, КАК делать задачу
✓ Понимание ЗАЧЕМ (смысл)
✓ Возможность влиять на решения
✓ Минимум контроля и микроменеджмента
✓ Признание чувств и перспектив
```

**Что разрушает автономию:**
```
✗ Директивный язык: "Ты должен", "Ты обязан"
✗ Жёсткие дедлайны без объяснения
✗ Постоянное наблюдение и отчёты
✗ Отсутствие выбора в методах работы
✗ Игнорирование мнения исполнителя
```

**Пример для разработчика:**
```
Низкая автономия:                 Высокая автономия:
─────────────────────             ────────────────────
"Сделай task #4521               "У нас проблема с
в точности как в ТЗ,             производительностью.
используй этот                   Как думаешь,
фреймворк, до пятницы"           что можно сделать?"

Результат: Выполню,              Результат: Включусь,
но без энтузиазма                предложу решения
```

### 2. Компетентность (Competence)

**Что это:** Ощущение, что ты способен, эффективен, можешь справляться с вызовами.

```
Компетентность ≠ Реальный навык

Это ОЩУЩЕНИЕ эффективности
Можно быть объективно компетентным, но не чувствовать этого
Можно чувствовать компетентность в процессе роста (даже если ещё не эксперт)
```

**Что поддерживает компетентность:**
```
✓ Оптимальный вызов (не слишком легко, не слишком сложно)
✓ Ясная, конструктивная обратная связь
✓ Возможность увидеть прогресс
✓ Признание достижений
✓ Возможности для развития
```

**Что разрушает компетентность:**
```
✗ Задачи слишком сложные (фрустрация) или слишком простые (скука)
✗ Отсутствие обратной связи
✗ Только негативная критика
✗ Сравнение с другими (а не с собой вчера)
✗ Непонятные критерии успеха
```

**Оптимальный вызов (Flow Channel):**

```
УРОВЕНЬ ВЫЗОВА
     │
 H   │              ТРЕВОГА
 I   │           ┌─────────────────
 G   │          ╱
 H   │         ╱   ЗОНА ПОТОКА
     │        ╱    (оптимальный вызов)
     │       ╱
     │      ╱
     │     ╱   СКУКА
 L   │    ╱
 O   │───╱─────────────────────────────
 W   │
     └────────────────────────────────►
           LOW                 HIGH
              УРОВЕНЬ НАВЫКА

Мотивация максимальна, когда вызов
соответствует текущему уровню навыка
```

### 3. Связность (Relatedness)

**Что это:** Ощущение связи с другими людьми, принадлежности, значимости для других.

```
Связность ≠ Экстраверсия

Интроверты тоже нуждаются в связности
Это не про количество общения, а про качество связей
Можно работать удалённо и чувствовать связность
```

**Что поддерживает связность:**
```
✓ Ощущение, что твоя работа важна для других
✓ Уважительные отношения в команде
✓ Чувство "мы делаем это вместе"
✓ Интерес коллег к тебе как к человеку
✓ Возможность помогать другим
```

**Что разрушает связность:**
```
✗ Изоляция без поддержки
✗ Токсичная атмосфера в команде
✗ Ощущение, что твоя работа никому не нужна
✗ Конкуренция вместо сотрудничества
✗ Отсутствие признания вклада
```

## Эффект сверхоправдания (Overjustification Effect)

### Открытие, изменившее понимание мотивации

Классический эксперимент Deci (1971):

```
ЭКСПЕРИМЕНТ С ПАЗЗЛАМИ SOMA

Группа A:                         Группа B:
─────────────────                 ─────────────────
Решают паззлы                     Решают паззлы
(без награды)                     (платят $1 за каждый)
        │                                 │
        ▼                                 ▼
Перерыв: наблюдают,               Перерыв: наблюдают,
что делают участники              что делают участники
        │                                 │
        ▼                                 ▼
Продолжают играть                 ПРЕКРАЩАЮТ играть
с паззлами добровольно            с паззлами
        │                                 │
        ▼                                 ▼
Внутренняя мотивация              Внутренняя мотивация
СОХРАНИЛАСЬ                       УМЕНЬШИЛАСЬ
```

### Почему это происходит

```
МЕХАНИЗМ СВЕРХОПРАВДАНИЯ

Ситуация:
"Мне нравится программировать"
(внутренняя мотивация)
            │
            ▼
Вводится награда:
"Теперь мне платят за каждую фичу"
            │
            ▼
Когнитивная переоценка:
"Почему я это делаю?"
            │
            ▼
Атрибуция смещается:
"Наверное, из-за денег"
(внешняя мотивация)
            │
            ▼
Внутренняя мотивация снижается:
"Без денег не буду делать"
```

**Ключевая идея:**

> Когда внешняя награда объясняет поведение, мозг перестаёт искать внутренние причины. Деятельность превращается из "хочу" в "надо ради награды".

### Когда награды особенно вредны

```
НАГРАДЫ СНИЖАЮТ ВНУТРЕННЮЮ МОТИВАЦИЮ, КОГДА:

1. Деятельность изначально интересна
   ✗ Платить за хобби
   ✗ Бонусы за креативные задачи

2. Награды ощущаются как контроль
   ✗ "Если сделаешь X — получишь Y"
   ✗ Награда обусловлена конкретным поведением

3. Награды ожидаемы заранее
   ✗ Система бонусов объявлена до работы
   ✗ Известно, что за это платят

4. Награды материальны и ощутимы
   ✗ Деньги, подарки
   (похвала вредит меньше)
```

### Когда награды НЕ вредят (и могут помочь)

```
НАГРАДЫ БЕЗОПАСНЫ ИЛИ ПОЛЕЗНЫ, КОГДА:

1. Деятельность изначально неинтересна
   ✓ Рутинные, механические задачи
   ✓ Работа, которую никто не хочет делать

2. Награды информационны (обратная связь)
   ✓ "Ты справился отлично" (компетентность)
   ✓ Неожиданная награда после работы

3. Награды поддерживают автономию
   ✓ "Ты сам решаешь, как использовать бонус"
   ✓ Признание, а не контроль

4. Награды символичны
   ✓ Признание перед командой
   ✓ Знак достижения (не ради денег)
```

## Применение в разработке ПО

### Почему open source работает

```
OPEN SOURCE МОТИВАЦИЯ (все три потребности)

АВТОНОМИЯ:
┌────────────────────────────────────────┐
│ • Выбираю проект сам                   │
│ • Решаю, над чем работать              │
│ • Могу уйти когда захочу               │
│ • Моё видение имеет вес                │
└────────────────────────────────────────┘

КОМПЕТЕНТНОСТЬ:
┌────────────────────────────────────────┐
│ • Вижу свой код в production           │
│ • Получаю code review от экспертов     │
│ • Учусь на реальных проектах           │
│ • Мой вклад виден публично             │
└────────────────────────────────────────┘

СВЯЗНОСТЬ:
┌────────────────────────────────────────┐
│ • Часть сообщества                     │
│ • Помогаю тысячам пользователей        │
│ • Коллаборация с единомышленниками     │
│ • Признание от комьюнити               │
└────────────────────────────────────────┘

Результат: Люди работают бесплатно
с бОльшим энтузиазмом, чем за зарплату
```

### Почему side projects мотивируют

```
SIDE PROJECT vs РАБОЧАЯ ЗАДАЧА

Side project:                     Рабочая задача:
─────────────────                 ─────────────────
Сам выбрал тему       АВТОНОМИЯ   Тему выбрал PM
Сам решаю как                     Есть требования
Нет дедлайна                      Жёсткие сроки

Учу новое             КОМПЕТЕНТ.  Использую известное
Вижу результат                    Результат где-то в монолите
Эксперименты ОК                   Ошибки = проблемы

Для себя/друзей      СВЯЗНОСТЬ    Для абстрактного пользователя
Могу показать                     NDA
Горжусь                           Просто работа

Энергии: МНОГО                    Энергии: МАЛО
(вечер после работы)              (8 часов рабочего дня)
```

### Почему некоторые команды мотивированы, а другие нет

```
МОТИВИРОВАННАЯ КОМАНДА:           ДЕМОТИВИРОВАННАЯ КОМАНДА:
═══════════════════════           ═════════════════════════

"Как думаете, как лучше?"         "Делай как я сказал"
        ↓                                 ↓
Автономия высокая                 Автономия низкая

Задачи чуть выше уровня           Либо рутина, либо аврал
Регулярный фидбек                 Фидбек только при ошибках
        ↓                                 ↓
Компетентность растёт             Компетентность стагнирует

"Мы делаем важную вещь"           "Это просто работа"
Уважение в команде                Токсичность/изоляция
        ↓                                 ↓
Связность высокая                 Связность низкая

═══════════════════════           ═════════════════════════
Результат: Энтузиазм              Результат: Выгорание
```

### Учёба: любопытство vs сертификат

```
ИЗУЧЕНИЕ ТЕХНОЛОГИИ

Ради любопытства:                 Ради сертификата:
─────────────────────             ─────────────────────
"Интересно, как это               "Нужно для резюме"
устроено"
        │                                 │
        ▼                                 ▼
Копаю глубоко                     Запоминаю для теста
Эксперименты                      Минимум для сдачи
Связываю с известным              Изолированные факты
        │                                 │
        ▼                                 ▼
Глубокое понимание                Поверхностное знание
Помню долго                       Забываю через месяц
Могу применить творчески          Могу применить по шаблону
```

## Снижение мотивации со временем

### Исследования в образовании

Мета-анализы показывают устойчивый паттерн:

```
МОТИВАЦИЯ В ОБРАЗОВАНИИ (типичная траектория)

Уровень │
мотива- │ ████
ции     │ ████████
        │ ████████████
        │ ████████████████
        │ ████████████████████
        │ ████████████████████████

        │ ████████████████████████████████
        │ ████████████████████████████████████
        └───────────────────────────────────────►
          Нач.   Средняя   Старшая    Универ-  Работа

```

### Почему это происходит

```
ФАКТОРЫ СНИЖЕНИЯ МОТИВАЦИИ:

1. Увеличение контроля
   Младшие классы: игра, исследование
   Старше: оценки, экзамены, требования

2. Снижение автономии
   "Учи что сказали, как сказали"

3. Социальное сравнение
   Конкуренция вместо сотрудничества

4. Потеря связи с интересами
   Абстрактные знания "для будущего"

5. Накопление негативного опыта
   Неудачи, критика, разочарования
```

### Это происходит и в карьере

```

        │       /  │  \/  │  \    /

        │     /    │      │    \/
        │    /     │      │
        │   /      │      │
        └──/───────┴──────┴─────────────────────►
         Первая   1-2     3-5    5-10+
         работа   года    лет    лет
```

## Интернализация мотивации

### Внешнее может стать внутренним

Хорошая новость: внешняя мотивация может трансформироваться во внутреннюю через процесс интернализации.

```
ПРОЦЕСС ИНТЕРНАЛИЗАЦИИ

ВНЕШНЯЯ РЕГУЛЯЦИЯ
"Делаю, потому что заставляют"
           │
           │ Понимание смысла
           ▼
ИНТРОЕЦИРОВАННАЯ РЕГУЛЯЦИЯ
"Делаю, чтобы не чувствовать вину/стыд"
           │
           │ Принятие ценности
           ▼
ИДЕНТИФИЦИРОВАННАЯ РЕГУЛЯЦИЯ
"Делаю, потому что это важно для моих целей"
           │
           │ Интеграция с идентичностью
           ▼
ИНТЕГРИРОВАННАЯ РЕГУЛЯЦИЯ
"Делаю, потому что это часть того, кто я есть"
           │
           │ (Внутренний интерес)
           ▼
ВНУТРЕННЯЯ МОТИВАЦИЯ
"Делаю, потому что это интересно само по себе"
```

### Что способствует интернализации

```
ДЛЯ ИНТЕРНАЛИЗАЦИИ НУЖНЫ:

1. ПОНИМАНИЕ СМЫСЛА (почему это важно)
   ─────────────────────────────────────
   ✗ "Пиши тесты, потому что так надо"
   ✓ "Тесты позволяют нам рефакторить без страха
      и выпускать быстрее"

2. АВТОНОМНАЯ ПОДДЕРЖКА (а не давление)
   ─────────────────────────────────────
   ✗ "Ты ДОЛЖЕН использовать этот паттерн"
   ✓ "Этот паттерн помогает в таких случаях.
      Что думаешь?"

3. СВЯЗЬ С ЦЕННОСТЯМИ (личная значимость)
   ─────────────────────────────────────
   ✗ "Документация — это требование"
   ✓ "Документация помогает будущим разработчикам
      (и тебе через полгода)"

4. КОМПЕТЕНТНОСТЬ (я могу это сделать)
   ─────────────────────────────────────
   ✗ Требовать без обучения
   ✓ Дать инструменты и поддержку
```

### Разница в переживании

```
"Я ДОЛЖЕН"                        "Я ХОЧУ"
(Внешняя / Интроецированная)      (Идентифицированная / Внутренняя)
═══════════════════════════       ═══════════════════════════════

"Я должен писать тесты,           "Я хочу писать тесты,
иначе не пройдёт PR"              потому что они дают уверенность"

"Я должен учить новое,            "Я хочу учить новое,
иначе отстану"                    потому что это расширяет возможности"

"Я должен помогать джуну,         "Я хочу помогать джуну,
это моя обязанность"              потому что помню себя и это важно"

        │                                 │
        ▼                                 ▼
Энергия тратится на                Энергия генерируется
преодоление сопротивления          от самой деятельности
```

## Burnout как истощение мотивации

### Связь SDT и выгорания

```
BURNOUT = ХРОНИЧЕСКОЕ НЕУДОВЛЕТВОРЕНИЕ БАЗОВЫХ ПОТРЕБНОСТЕЙ

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   АВТОНОМИЯ ↓        КОМПЕТЕНТНОСТЬ ↓      СВЯЗНОСТЬ ↓     │
│   ───────────        ───────────────       ──────────       │
│   Микроменеджмент    Невозможные сроки     Изоляция         │
│   Нет влияния        Нет обратной связи    Токсичность      │
│   Бессмысленность    Только критика        Одиночество      │
│                                                             │
│                         │                                   │
│                         ▼                                   │
│              ┌─────────────────────┐                        │
│              │   ИСТОЩЕНИЕ         │                        │
│              │   ЦИНИЗМ            │                        │
│              │   НЕЭФФЕКТИВНОСТЬ   │                        │
│              └─────────────────────┘                        │
│                      BURNOUT                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Профилактика через SDT

```
ВОССТАНОВЛЕНИЕ / ПРОФИЛАКТИКА

Симптом burnout:             SDT-интервенция:
────────────────────         ─────────────────────────────
"Всё бессмысленно"           Найти/вспомнить ЗАЧЕМ
(потеря смысла)              Связать работу с ценностями
                             Автономия в выборе задач

"Я ни на что                 Маленькие достижимые цели
не способен"                 Позитивная обратная связь
(потеря компетентности)      Зона ближайшего развития

"Никому не нужен"            Восстановить связи
(потеря связности)           Найти единомышленников
                             Менторство (в обе стороны)
```

## Как поддерживать мотивацию

### Чек-лист поддержки автономии

```
ДЛЯ СЕБЯ:
─────────
□ Понимаю ли я ЗАЧЕМ делаю эту задачу?
□ Есть ли у меня выбор в том, КАК её делать?
□ Могу ли я влиять на приоритеты?
□ Работаю ли я над тем, что считаю важным?
□ Есть ли у меня время на собственные инициативы?

ДЛЯ КОМАНДЫ (если ты лид):
──────────────────────────
□ Объясняю ли я смысл задач, а не только что делать?
□ Даю ли я выбор в способе выполнения?
□ Спрашиваю ли мнение до принятия решений?
□ Избегаю ли контролирующего языка?
□ Признаю ли право на несогласие?
```

### Чек-лист поддержки компетентности

```
ДЛЯ СЕБЯ:
─────────
□ Задачи в зоне оптимального вызова?
□ Получаю ли я регулярную обратную связь?
□ Вижу ли я свой прогресс?
□ Есть ли возможности для развития?
□ Отмечаю ли я свои достижения?

ДЛЯ КОМАНДЫ:
────────────
□ Подбираю ли задачи по уровню?
□ Даю ли конструктивную обратную связь?
□ Признаю ли достижения публично?
□ Создаю ли возможности для роста?
□ Помогаю ли видеть прогресс?
```

### Чек-лист поддержки связности

```
ДЛЯ СЕБЯ:
─────────
□ Чувствую ли я связь с командой?
□ Понимаю ли, как моя работа влияет на других?
□ Есть ли у меня значимые профессиональные отношения?
□ Помогаю ли я другим?
□ Участвую ли в комьюнити?

ДЛЯ КОМАНДЫ:
────────────
□ Создаю ли я атмосферу психологической безопасности?
□ Показываю ли связь работы с влиянием на пользователей?
□ Поддерживаю ли сотрудничество, а не конкуренцию?
□ Знаю ли я людей как людей, а не только как работников?
□ Есть ли традиции, укрепляющие команду?
```

### Что избегать

```
РАЗРУШИТЕЛИ МОТИВАЦИИ:

КОНТРОЛИРУЮЩИЙ ЯЗЫК:
✗ "Ты должен", "Ты обязан"
✗ "Сделай это правильно"
✗ "Почему ты не сделал как я сказал?"

ПОСТОЯННОЕ НАБЛЮДЕНИЕ:
✗ Требование частых отчётов без причины
✗ Отслеживание времени до минуты
✗ Контроль процесса, а не результата

ВНЕШНИЕ ОЦЕНКИ БЕЗ ОБРАТНОЙ СВЯЗИ:
✗ Рейтинги без объяснения
✗ Сравнение с коллегами
✗ Публичная критика

НАГРАДЫ ЗА ИНТЕРЕСНУЮ РАБОТУ:
✗ Бонусы за креативные задачи
✗ "Если сделаешь хорошо — получишь X"
✗ Превращение хобби в обязанность

ИГНОРИРОВАНИЕ БАЗОВЫХ ПОТРЕБНОСТЕЙ:
✗ Навязывание без объяснения (автономия)
✗ Невозможные требования (компетентность)
✗ Изоляция и токсичность (связность)
```

## Практические стратегии

### Восстановление внутренней мотивации

```
ЕСЛИ МОТИВАЦИЯ ПОТЕРЯНА:

1. ВСПОМНИ ЗАЧЕМ
   ─────────────────────────────────────
   • Почему ты изначально выбрал эту работу/проект?
   • Какую ценность это создаёт?
   • Кому это помогает?

2. ВЕРНИ АВТОНОМИЮ
   ─────────────────────────────────────
   • Найди область, где можешь выбирать
   • Возьми side project для себя
   • Предложи своё решение вместо ожидания указаний

3. СОЗДАЙ МАЛЕНЬКИЕ ПОБЕДЫ
   ─────────────────────────────────────
   • Разбей большую задачу на видимые шаги
   • Отмечай завершение каждого шага
   • Визуализируй прогресс (чек-листы, kanban)

4. ВОССТАНОВИ СВЯЗИ
   ─────────────────────────────────────
   • Поговори с коллегами о чём-то кроме работы
   • Найди ментора или стань ментором
   • Присоединись к комьюнити
```

### Поддержание мотивации в долгосроке

```
УСТОЙЧИВЫЕ ПРАКТИКИ:

ЕЖЕДНЕВНО:
• Начни день с задачи, которая нравится
• Найди хотя бы один момент "потока"
• Замечай маленькие достижения

ЕЖЕНЕДЕЛЬНО:
• Рефлексия: что давало энергию, что забирало?
• Один час на обучение/эксперименты
• Общение с единомышленниками

ЕЖЕМЕСЯЧНО:
• Оценка: удовлетворены ли три потребности?
• Корректировка: что можно изменить?
• Планирование: что интересного впереди?

ЕЖЕГОДНО:
• Глубокая ревизия: туда ли иду?
• Обновление целей и смыслов
• Предотвращение выгорания
```

### Мотивация в обучении

```
КАК УЧИТЬСЯ, СОХРАНЯЯ МОТИВАЦИЮ:

1. ВЫБИРАЙ САМ (автономия)
   • Тему, которая реально интересует
   • Формат, который подходит (видео/текст/практика)
   • Темп, который комфортен

2. КАЛИБРУЙ СЛОЖНОСТЬ (компетентность)
   • Начни с того, что можешь понять
   • Постепенно усложняй
   • Не перепрыгивай базу

3. ПРИМЕНЯЙ (компетентность + связность)
   • Реальные проекты > абстрактные упражнения
   • Покажи кому-то результат
   • Помоги другому с тем, что узнал

4. ИЗБЕГАЙ ВНЕШНИХ ДАВЛЕНИЙ
   • Учись не ради сертификата, а ради понимания
   • Не сравнивай себя с другими
   • Разреши себе учиться медленно
```

## Actionable: что делать

### На этой неделе:

```
□ Выбери одну задачу и спроси себя: "Зачем я это делаю?"
  Если нет ответа — найди смысл или пересмотри приоритет

□ Найди зону автономии: хотя бы одну вещь,
  которую делаешь по своему выбору

□ Отметь три маленьких достижения за неделю
  (буквально запиши)

□ Поговори с коллегой о чём-то кроме работы
```

### В этом месяце:

```
□ Проведи аудит трёх потребностей:
  - Автономия: где я выбираю / где нет?
  - Компетентность: расту ли я / в чём застрял?
  - Связность: с кем связан / где одинок?

□ Начни side project (без внешних целей — только интерес)

□ Если чувствуешь демотивацию — определи,
  какая потребность не удовлетворена
```

### В долгосроке:

```
□ Регулярно пересматривай: работа даёт или забирает энергию?

□ Строй карьеру вокруг внутренней мотивации,
  не только внешних наград

□ Создавай среду, поддерживающую три потребности
  (для себя и команды)

□ Помни: мотивация — не бесконечный ресурс,
  её нужно восстанавливать
```
