# Шаблон материала по Tech Leadership

> **Этот шаблон обязателен для КАЖДОГО материала в leadership/.**
> Адаптирован из template-communication.md для технического лидерства.

---

## Принципы

1. **Практический фокус** — конкретные действия, не абстрактные советы
2. **Реальные примеры** — из опыта Google, Meta, Amazon, Netflix, стартапов
3. **Исследования** — данные из Gallup, Harvard, Google rework, McKinsey
4. **Скрипты и templates** — готовые фразы для 1-on-1, найма, feedback
5. **Trade-offs** — у каждого подхода есть плюсы и минусы

## Структура материала (12 секций)

### 1. TL;DR (3-5 предложений)

```markdown
> **TL;DR:** [Роль/навык] — это [определение].
> Ключевое отличие от [смежной роли]: [в чём разница].
> Главная ответственность: [1 предложение].
> Типичная ошибка: [1 предложение].
```

### 2. Зачем это нужно? (Проблема + статистика)

```markdown
## Зачем это нужно?

### Типичная ситуация

[Конкретный сценарий: первые 90 дней, найм, сложный разговор]

**Без понимания [темы]:**
[Что происходит — плохие решения, выгорание, текучка]

**С пониманием [темы]:**
[Результат — эффективность, рост команды, delivery]

### Статистика

| Метрика | Значение | Источник |
|---------|----------|----------|
| [Метрика 1] | [Число] | [Gallup/HBR/Google] |
| [Метрика 2] | [Число] | [Источник] |
```

### 7. Скрипты и Templates

```markdown
### Сценарий 1: [Название]

```
"[Готовая фраза]"
"[Альтернативная формулировка]"
```

**Не говори:**
```
"[Плохой вариант]" → потому что [причина]
```

### Template: [Название]

```
[Шаблон документа/email/agenda]
```

### Чеклист: [Название]

□ [Пункт с действием]
□ [Пункт с действием]
```

**Минимум:** 5 скриптов, 3 templates

## Чеклист качества материала

```
ИССЛЕДОВАНИЕ:
□ Awesome-lists проверены на релевантные ресурсы
□ 15+ источников проанализированы
□ Примеры из реальных компаний включены

СТРУКТУРА:
□ 12 обязательных секций присутствуют
□ 2500-4500 слов
□ ASCII-визуализации для моделей
□ Таблицы для сравнений

СОДЕРЖАНИЕ:
□ Минимум 5 скриптов/ready-to-use фраз
□ Минимум 3 templates (documents, agendas)
□ Минимум 2 кейса из реальных компаний
□ Trade-offs и "когда НЕ использовать" раскрыты
□ Статистика с источниками

КАЧЕСТВО ТЕКСТА:
□ Нет AI-маркеров (delve, robust, crucial)
□ Конкретика вместо общих слов
□ Actionable advice в каждой секции
□ Прямое обращение к читателю

СВЯЗИ:
□ Prerequisites указаны как [[ссылки]]
□ Unlocks указаны
□ Связи с communication/ и career/
```
