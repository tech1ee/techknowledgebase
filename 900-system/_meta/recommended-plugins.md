# Рекомендуемые плагины для обучения

Эта база знаний работает как обычный Obsidian-вольт: wiki-ссылки, frontmatter, callout-блоки, flashcard-разметка -- всё читается без единого плагина. Но 4-7 плагинов превращают статичный вольт в систему обучения с интервальным повторением, отслеживанием прогресса и автоматической генерацией карточек. Ключевой инсайт: плагины не добавляют контент -- они активируют то, что уже заложено в структуру файлов.

---

## Сводная таблица

| Плагин | Назначение | Приоритет | Mobile |
|--------|-----------|:---------:|:------:|
| Dataview | Дашборды, отслеживание прогресса | Tier 1 | частично |
| Spaced Repetition | Интервальное повторение (SM-2) | Tier 1 | да |
| Templater | Автоматизация frontmatter | Tier 2 | нет |
| Smart Connections | Семантический поиск, AI-чат | Tier 2 | нет |
| Quiz Generator | AI-генерация тестов из заметок | Tier 3 | нет |
| Calendar | Визуальная лента активности | Tier 3 | да |
| Obsidian Git | Синхронизация через GitHub | Tier 3 | Android |

---

## Tier 1: Обязательные

Эти два плагина раскрывают основные функции вольта -- дашборд прогресса и интервальное повторение. Без них файлы остаются текстом; с ними -- становятся системой.

### Dataview

Dataview превращает frontmatter-метаданные в SQL-подобные запросы прямо внутри заметок. Вместо того чтобы вручную искать файлы по статусу, уровню или дате, ты пишешь запрос -- и получаешь живую таблицу, которая обновляется при каждом открытии.

**Зачем именно этому вольту.** Каждый файл содержит YAML-поля: `type`, `status`, `confidence`, `tags`, `created`, `modified`. Файл `study-dashboard.md` построен на Dataview-запросах -- без плагина он показывает сырой код вместо таблиц. Dataview позволяет создавать обзоры вроде "все файлы по Android с confidence: outdated" или "что изменилось за последнюю неделю" -- без ручного обхода 750 файлов.

**Установка:** Community Plugins > Browse > "Dataview" (автор: Michael Brenan).

**Настройки после установки:**
- `Enable JavaScript Queries` -- включить, если планируешь сложные запросы (dataviewjs)
- `Enable Inline Queries` -- включить для inline-запросов в тексте (`` `= this.file.name` ``)
- `Refresh Interval` -- оставить 2500ms (по умолчанию), на больших вольтах повышает если тормозит

**Пример запроса для этого вольта:**

```dataview
TABLE type, status, confidence
FROM "200-platforms/android"
WHERE status = "published"
SORT modified DESC
LIMIT 15
```

Этот запрос выведет 15 последних изменённых опубликованных файлов по Android -- с типом, статусом и уровнем уверенности. Dataview читает YAML frontmatter напрямую, поэтому все поля из [[frontmatter-standard]] доступны как колонки.

### Spaced Repetition

Плагин реализует алгоритм SM-2 (SuperMemo 2) для интервального повторения карточек прямо внутри Obsidian. Карточки не нужно создавать отдельно -- плагин находит их в обычных markdown-файлах по специальному разделителю.

**Зачем именно этому вольту.** Файлы вольта содержат блоки в формате flashcard:

```
Вопрос
?
Ответ
```

Плагин Spaced Repetition распознаёт этот синтаксис и строит из него колоду карточек с расписанием повторений. Без плагина эти блоки читаются как обычный текст (вопрос-ответ) -- полезно, но без scheduling. С плагином -- полноценный Anki-подобный цикл повторений, встроенный в вольт. Подробности настройки: [[spaced-repetition-guide]].

**Установка:** Community Plugins > Browse > "Spaced Repetition" (автор: st3v3nmw).

**Настройки после установки:**
- `Flashcard tags` -- добавить `#flashcards` (плагин ищет карточки только в файлах с этим тегом, либо настроить на все файлы)
- `Separator for inline flashcards` -- оставить `?` (это разделитель, используемый в вольте)
- `Convert folders to decks` -- включить; папки `200-platforms/android/`, `200-platforms/ios/` автоматически станут отдельными колодами
- `Bury sibling cards` -- включить, чтобы карточки из одного файла не шли подряд (spacing effect)
- `New card insertion` -- выбрать `Uniform` для равномерного распределения новых карточек

**Рабочий цикл:** Открываешь боковую панель плагина > выбираешь колоду (или "All") > карточки показываются по расписанию SM-2. Оцениваешь: Hard / Good / Easy -- интервал до следующего показа рассчитывается автоматически. Метаданные повторений хранятся в HTML-комментариях внутри файла, не ломая frontmatter.

## Tier 2: Рекомендуемые

Не критичны для базовой работы, но заметно ускоряют рутинные операции и открывают неочевидные связи между темами.

### Templater

Templater заменяет встроенные шаблоны Obsidian на систему с переменными, условиями и JavaScript-вставками. Главная задача -- автоматически генерировать frontmatter при создании новой заметки, не заполняя 8 полей вручную.

**Зачем именно этому вольту.** Стандарт frontmatter ([[frontmatter-standard]]) требует `title`, `created`, `modified`, `type`, `tags`, и опционально `status`, `confidence`, `related`. Templater подставляет дату автоматически, предлагает выбор `type` из списка допустимых значений, и формирует tags на основе папки, в которой создаётся файл. Это снижает вероятность ошибок в метаданных и экономит 30-60 секунд на каждой заметке.

**Установка:** Community Plugins > Browse > "Templater" (автор: SilentVoid).

**Настройки после установки:**
- `Template folder location` -- указать папку с шаблонами (например, `900-system/_templates/`)
- `Trigger Templater on new file creation` -- включить
- `Enable folder templates` -- включить; назначить разные шаблоны для `200-platforms/android/`, `200-platforms/ios/` и т.д.

**Пример шаблона для этого вольта:**

```markdown
---
title: "<% tp.file.cursor(1) %>"
created: <% tp.date.now("YYYY-MM-DD") %>
modified: <% tp.date.now("YYYY-MM-DD") %>
type: <% tp.system.suggester(["concept","guide","reference","tutorial","overview"], ["concept","guide","reference","tutorial","overview"]) %>
status: draft
confidence: high
tags:
  - topic/<% tp.file.folder(true).split("/").pop() %>
---
```

### Smart Connections

Smart Connections строит векторные эмбеддинги для каждой заметки и находит семантически похожие файлы, даже если между ними нет явных wiki-ссылок. Плагин также предоставляет AI-чат, который отвечает на вопросы по содержимому вольта.

**Зачем именно этому вольту.** В 750 файлах по 20+ доменам (Android, iOS, KMP, архитектура, безопасность, сети) неизбежно есть пересечения, которые не покрыты ручными ссылками. Например, файл о `android-networking.md` семантически связан с `network-http-evolution.md` и `ios-networking.md` -- но ссылки между ними может не быть. Smart Connections находит эти связи автоматически. AI-чат позволяет задать вопрос вроде "Чем отличается обработка фоновых задач в Android и iOS?" -- и получить ответ, синтезированный из нескольких файлов вольта.

**Установка:** Community Plugins > Browse > "Smart Connections" (автор: Brian Petro).

**Настройки после установки:**
- `Embedding model` -- для локальной работы выбрать `transformers` (бесплатно, офлайн); для лучшего качества -- OpenAI `text-embedding-3-small` (нужен API-ключ)
- `Excluded folders` -- добавить `900-system/_meta/`, `docs/` (системные файлы не нужны в семантическом поиске)
- `Force refresh` -- запустить после первого включения для построения индекса (занимает 3-10 минут на 750 файлов)

## Tier 3: Опциональные

Полезные дополнения, которые решают конкретные задачи. Ставить по необходимости.

### Quiz Generator

Quiz Generator создаёт тесты (multiple choice, true/false, open-ended) из содержимого любой заметки с помощью LLM. Поддерживает OpenAI, Google Gemini и локальные модели через Ollama.

**Зачем именно этому вольту.** Вольт содержит объёмные концептуальные файлы (3000-8000 слов) по архитектуре, системам, алгоритмам. Flashcard-блоки покрывают ключевые факты, но не проверяют понимание связей и применение. Quiz Generator генерирует вопросы более высокого уровня по Bloom's Taxonomy -- анализ, оценка, применение -- что дополняет SM-2 карточки.

**Установка:** Community Plugins > Browse > "Quiz Generator" (автор: ECuiDev).

**Настройки:** Указать провайдера (OpenAI / Gemini / Ollama) и API-ключ. Для Ollama ключ не нужен, но требуется локально запущенная модель. Количество вопросов и язык настраиваются в параметрах плагина -- указать русский.

### Calendar

Calendar отображает календарную сетку с точками на днях, когда создавались или изменялись заметки. Клик по дню открывает daily note.

**Зачем именно этому вольту.** Визуализирует регулярность работы с базой знаний. Поле `modified` в frontmatter обновляется вручную, но Calendar привязывается к системной дате файла. Помогает отследить пробелы в активности и поддерживать streak.

**Установка:** Community Plugins > Browse > "Calendar" (автор: Liam Cain).

### Obsidian Git

Obsidian Git автоматически делает commit и push в GitHub-репозиторий по расписанию (например, каждые 30 минут) или при закрытии приложения.

**Зачем именно этому вольту.** Вольт хранится в публичном GitHub-репозитории. Без плагина нужно вручную открывать терминал и делать `git add / commit / push`. Obsidian Git автоматизирует это -- изменения попадают в репозиторий без выхода из Obsidian. На Android работает через isomorphic-git (без нативного git), на iOS не поддерживается.

**Установка:** Community Plugins > Browse > "Obsidian Git" (автор: Vinzent).

**Настройки:** `Auto pull interval` -- 10 минут, `Auto push interval` -- 10 минут, `Commit message` -- "vault backup: {{date}}". Убедиться, что `.obsidian/` в `.gitignore` (в этом вольте уже настроено).

## Без плагинов

Плагины расширяют возможности, но вольт спроектирован так, чтобы ядро работало и без них. Вот что доступно в чистом Obsidian:

**Навигация.** Wiki-ссылки (`[[файл]]`) работают нативно. Структура Home.md > MOC > Overview > Content файлы полностью кликабельна без плагинов. Graph View показывает связи между файлами.

**Frontmatter.** Obsidian отображает YAML-метаданные в панели Properties (начиная с v1.4). Поля `title`, `type`, `tags`, `status` видны и редактируемы через GUI -- Dataview не нужен для просмотра, только для запросов.

**Flashcard-блоки.** Формат `Вопрос\n?\nОтвет` читается как обычный текст. Без плагина Spaced Repetition нет автоматического расписания повторений, но сами вопросы и ответы полностью функциональны для ручного self-testing.

**Callout-блоки.** Блоки `> [!tip]`, `> [!warning]`, `> [!info]` -- встроенная функция Obsidian, рендерятся без плагинов.

**Поиск.** Встроенный поиск Obsidian поддерживает regex, фильтрацию по пути (`path:`), тегам (`tag:`), и содержимому YAML-полей (`[type:guide]`). Для базовых задач Smart Connections не обязателен.

## Порядок установки

Рекомендуемая последовательность учитывает зависимости и приоритет:

1. **Spaced Repetition** -- сразу запускает цикл повторений. Flashcard-блоки уже есть в файлах; плагин начинает работать немедленно.
2. **Dataview** -- активирует дашборд прогресса и позволяет строить обзоры по метаданным.
3. **Templater** -- настроить после того, как понимаешь структуру frontmatter. Шаблоны создаются один раз, дальше работают автоматически.
4. **Smart Connections** -- индексация требует времени. Поставить и дать проиндексировать вольт в фоне.
5. **Calendar / Quiz Generator / Obsidian Git** -- по необходимости, в любом порядке.

> [!tip] Между Tier 1 и Tier 2 -- пауза
> Поработай неделю только со Spaced Repetition и Dataview. Убедись, что цикл повторений настроен, дашборд показывает то, что нужно. Только потом добавляй Tier 2 -- иначе настройка 4 плагинов одновременно перегрузит (cognitive load, 4±1 элемент).
