# Communication MOC

> Коммуникационные навыки для IT-специалистов: от моделей и слушания до переговоров, презентаций и кросс-культурной работы.

---

## Статьи по категориям

### Fundamentals
- [[communication-models]] — модели коммуникации (Shannon-Weaver, транзакционная, SMCR) и диагностика проблем
- [[communication-barriers]] — типы барьеров (physical, semantic, psychological, cultural) и способы их преодоления
- [[communication-styles]] — модель DISC и адаптация стиля общения к собеседнику

### Listening
- [[active-listening]] — техника полной концентрации на говорящем: парафраз, уточняющие вопросы, SOLER
- [[empathetic-listening]] — эмоциональная настройка и validation чувств собеседника без согласия/несогласия

### Feedback
- [[giving-feedback]] — SBI-модель, Radical Candor и структурированная обратная связь для code review и 1-on-1
- [[receiving-feedback]] — управление defensiveness, growth mindset и формула: пауза -> уточнение -> благодарность -> действие
- [[feedback-frameworks]] — сравнение SBI, COIN, DESC, STAR, CEDAR и выбор фреймворка под ситуацию

### Difficult Conversations
- [[conflict-resolution]] — Thomas-Kilmann 5 стилей, interest-based resolution и disagree-and-commit
- [[performance-conversations]] — структура performance reviews, PIP и promotion discussions без разрушения отношений
- [[saying-no]] — профессиональный отказ: Acknowledge -> Decline -> Reason -> Alternative
- [[delivering-bad-news]] — сообщение о layoffs, cancellations и кризисных ситуациях с сохранением доверия

### Negotiation
- [[negotiation-fundamentals]] — BATNA, ZOPA, anchoring и Harvard principled negotiation для IT-контекста
- [[negotiation-frameworks]] — Harvard Method, integrative bargaining и win-win стратегии
- [[stakeholder-negotiation]] — влияние без authority: работа с executives, PM и клиентами

### Presentations
- [[technical-presentations]] — Minto Pyramid, структура tech talk и техники демонстрации
- [[presentation-design]] — визуальный дизайн слайдов, правило 10-20-30 и один слайд = одна идея
- [[storytelling-tech]] — нарративные техники для IT: Hero's Journey для кода и data storytelling

### Written Communication
- [[email-communication]] — BLUF (Bottom Line Up Front) и 5Cs framework для эффективных email
- [[technical-writing]] — RFC, ADR и документация: контекст + решение + обоснование
- [[async-communication]] — async-first подход для remote-команд: Slack, email и best practices

### Cross-Cultural Communication
- [[cultural-dimensions]] — модели Hofstede и Lewis для понимания культурных различий в глобальных командах
- [[remote-team-communication]] — async-first, overlap hours, explicit communication и building trust
