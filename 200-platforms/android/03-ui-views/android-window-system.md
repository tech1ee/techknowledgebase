# Window System: PhoneWindow, DecorView, WindowManager –Є Surface

> –Ъ–∞–ґ–і—Л–є –њ–Є–Ї—Б–µ–ї—М –љ–∞ —Н–Ї—А–∞–љ–µ Android –њ—А–Є–љ–∞–і–ї–µ–ґ–Є—В –Ї–∞–Ї–Њ–Љ—Г-—В–Њ –Њ–Ї–љ—Г (Window). Activity, Dialog, Toast, –Ї–ї–∞–≤–Є–∞—В—Г—А–∞, —Б—В–∞—В—Г—Б-–±–∞—А вАФ –≤—Б–µ —Н—В–Њ –Њ–Ї–љ–∞, —Г–њ—А–∞–≤–ї—П–µ–Љ—Л–µ WindowManagerService –≤ system_server –Є –Њ—В–Њ–±—А–∞–ґ–∞–µ–Љ—Л–µ —З–µ—А–µ–Ј SurfaceFlinger. Window System вАФ —Н—В–Њ –Љ–љ–Њ–≥–Њ—Г—А–Њ–≤–љ–µ–≤–∞—П –∞—А—Е–Є—В–µ–Ї—В—Г—А–∞: –њ—А–Є–ї–Њ–ґ–µ–љ–Є–µ —А–∞–±–Њ—В–∞–µ—В —Б –∞–±—Б—В—А–∞–Ї—В–љ—Л–Љ Window API, ViewRootImpl —В—А–∞–љ—Б–ї–Є—А—Г–µ—В —Н—В–Њ –≤ Binder IPC –Ї WMS, –∞ SurfaceFlinger –Ї–Њ–Љ–њ–Њ–Ј–Є—В–Є—В –≤—Б–µ Surface –≤ —Д–Є–љ–∞–ї—М–љ–Њ–µ –Є–Ј–Њ–±—А–∞–ґ–µ–љ–Є–µ —Б–Є–љ—Е—А–Њ–љ–љ–Њ —Б VSYNC. –Я–Њ–љ–Є–Љ–∞–љ–Є–µ —Н—В–Њ–є —Б–Є—Б—В–µ–Љ—Л –Ї—А–Є—В–Є—З–љ–Њ –і–ї—П edge-to-edge –і–Є–Ј–∞–є–љ–∞, –њ—А–∞–≤–Є–ї—М–љ–Њ–є —А–∞–±–Њ—В—Л —Б insets –Є –і–Є–∞–≥–љ–Њ—Б—В–Є–Ї–Є rendering issues.

## –Ґ–µ–Њ—А–µ—В–Є—З–µ—Б–Ї–Є–µ –Њ—Б–љ–Њ–≤—Л

> **Window system** (–Њ–Ї–Њ–љ–љ–∞—П —Б–Є—Б—В–µ–Љ–∞) вАФ –Ї–Њ–Љ–њ–Њ–љ–µ–љ—В –Ю–°, —Г–њ—А–∞–≤–ї—П—О—Й–Є–є –≤—Л–і–µ–ї–µ–љ–Є–µ–Љ –Њ–±–ї–∞—Б—В–µ–є —Н–Ї—А–∞–љ–∞ –і–ї—П –њ—А–Є–ї–Њ–ґ–µ–љ–Є–є –Є –Є—Е –Ї–Њ–Љ–њ–Њ–Ј–Є—Ж–Є–µ–є –≤ —Д–Є–љ–∞–ї—М–љ–Њ–µ –Є–Ј–Њ–±—А–∞–ґ–µ–љ–Є–µ. –Ъ–Њ–љ—Ж–µ–њ—Ж–Є—П —Д–Њ—А–Љ–∞–ї–Є–Ј–Њ–≤–∞–љ–∞ –≤ X Window System (Scheifler/Gettys, MIT, 1984): —А–∞–Ј–і–µ–ї–µ–љ–Є–µ **window manager** (—Г–њ—А–∞–≤–ї–µ–љ–Є–µ –Њ–Ї–љ–∞–Љ–Є) –Є **display server** (–Њ—В—А–Є—Б–Њ–≤–Ї–∞).

Android Window System —А–µ–∞–ї–Є–Ј—Г–µ—В –∞—А—Е–Є—В–µ–Ї—В—Г—А—Г **compositing window manager**, –≥–і–µ –Ї–∞–ґ–і–Њ–µ –њ—А–Є–ї–Њ–ґ–µ–љ–Є–µ —А–Є—Б—Г–µ—В –≤ —Б–Њ–±—Б—В–≤–µ–љ–љ—Л–є –±—Г—Д–µ—А (Surface), –∞ —Б–Є—Б—В–µ–Љ–љ—Л–є –њ—А–Њ—Ж–µ—Б—Б (SurfaceFlinger) —Б–Њ–±–Є—А–∞–µ—В –≤—Б–µ –±—Г—Д–µ—А—Л –≤ —Д–Є–љ–∞–ї—М–љ—Л–є –Ї–∞–і—А. –≠—В–∞ –∞—А—Е–Є—В–µ–Ї—В—Г—А–∞ —Б—В–∞–ї–∞ —Б—В–∞–љ–і–∞—А—В–Њ–Љ –њ–Њ—Б–ї–µ Compiz (Linux, 2006), Quartz Compositor (macOS, 2001) –Є DWM (Windows Vista, 2006).

| –Ъ–Њ–Љ–њ–Њ–љ–µ–љ—В | –Я—А–Њ—Ж–µ—Б—Б | –£—А–Њ–≤–µ–љ—М –∞–±—Б—В—А–∞–Ї—Ж–Є–Є | –Р–љ–∞–ї–Њ–≥ –≤ X11/Wayland |
|-----------|---------|-------------------|---------------------|
| PhoneWindow | App process | –Т—Л—Б–Њ–Ї–Є–є (Java API) | Window (Xlib) |

| WindowManagerService | system_server | –°–Є—Б—В–µ–Љ–љ—Л–є (–њ–Њ–ї–Є—В–Є–Ї–∞) | Window Manager (Mutter, KWin) |
| SurfaceFlinger | –Ю—В–і–µ–ї—М–љ—Л–є native –њ—А–Њ—Ж–µ—Б—Б | –Э–Є–Ј–Ї–Є–є (compositor) | Display Server (Xorg, Wayland) |
| HWC (Hardware Composer) | HAL | –Р–њ–њ–∞—А–∞—В–љ—Л–є | DRM/KMS |

> **–Ф–≤–Њ–є–љ–∞—П/—В—А–Њ–є–љ–∞—П –±—Г—Д–µ—А–Є–Ј–∞—Ж–Є—П** (double/triple buffering) вАФ —В–µ—Е–љ–Є–Ї–∞ —Г—Б—В—А–∞–љ–µ–љ–Є—П **screen tearing**, —Д–Њ—А–Љ–∞–ї–Є–Ј–Њ–≤–∞–љ–љ–∞—П –≤ –Ї–Њ–љ—В–µ–Ї—Б—В–µ —А–∞—Б—В—А–Њ–≤—Л—Е –і–Є—Б–њ–ї–µ–µ–≤ (Sproull/Sutherland, 1968). –Я—А–Є–ї–Њ–ґ–µ–љ–Є–µ —А–Є—Б—Г–µ—В –≤ back buffer, –∞ –і–Є—Б–њ–ї–µ–є —З–Є—В–∞–µ—В –Є–Ј front buffer. –Я–µ—А–µ–Ї–ї—О—З–µ–љ–Є–µ –±—Г—Д–µ—А–Њ–≤ –њ—А–Њ–Є—Б—Е–Њ–і–Є—В –∞—В–Њ–Љ–∞—А–љ–Њ –њ–Њ —Б–Є–≥–љ–∞–ї—Г VSYNC.

**BufferQueue** –≤ Android вАФ —А–µ–∞–ї–Є–Ј–∞—Ж–Є—П –Ї–ї–∞—Б—Б–Є—З–µ—Б–Ї–Њ–є –Ј–∞–і–∞—З–Є **Producer-Consumer** (Dijkstra, 1965): –њ—А–Є–ї–Њ–ґ–µ–љ–Є–µ (producer) –Ј–∞–њ–Њ–ї–љ—П–µ—В –≥—А–∞—Д–Є—З–µ—Б–Ї–Є–µ –±—Г—Д–µ—А—Л, SurfaceFlinger (consumer) –Ј–∞–±–Є—А–∞–µ—В –Є—Е –і–ї—П –Ї–Њ–Љ–њ–Њ–Ј–Є—В–Є–љ–≥–∞. –°–Є–љ—Е—А–Њ–љ–Є–Ј–∞—Ж–Є—П —З–µ—А–µ–Ј **fence** вАФ GPU-–њ—А–Є–Љ–Є—В–Є–≤, —Б–Є–≥–љ–∞–ї–Є–Ј–Є—А—Г—О—Й–Є–є –Њ –Ј–∞–≤–µ—А—И–µ–љ–Є–Є –Ј–∞–њ–Є—Б–Є –≤ –±—Г—Д–µ—А.

PhoneWindow вЖТ DecorView вЖТ ContentView вАФ —Ж–µ–њ–Њ—З–Ї–∞, —А–µ–∞–ї–Є–Ј—Г—О—Й–∞—П –њ–∞—В—В–µ—А–љ **Decorator** (Gamma et al., 1994): –Ї–∞–ґ–і—Л–є —Г—А–Њ–≤–µ–љ—М –і–Њ–±–∞–≤–ї—П–µ—В —Д—Г–љ–Ї—Ж–Є–Њ–љ–∞–ї—М–љ–Њ—Б—В—М (—Б–Є—Б—В–µ–Љ–љ—Л–µ –±–∞—А—Л, –Ј–∞–≥–Њ–ї–Њ–≤–Њ–Ї, —В–µ–Љ–∞) –Ї –њ–Њ–ї—М–Ј–Њ–≤–∞—В–µ–ї—М—Б–Ї–Њ–Љ—Г layout –±–µ–Ј –µ–≥–Њ –Љ–Њ–і–Є—Д–Є–Ї–∞—Ж–Є–Є. `setContentView()` –њ–Њ–Љ–µ—Й–∞–µ—В layout —А–∞–Ј—А–∞–±–Њ—В—З–Є–Ї–∞ –≤ `android.R.id.content` FrameLayout –≤–љ—Г—В—А–Є DecorView.

> **Insets** вАФ —Д–Њ—А–Љ–∞–ї—М–љ–∞—П –Љ–Њ–і–µ–ї—М **–њ—А–Њ—Б—В—А–∞–љ—Б—В–≤–µ–љ–љ—Л—Е –Њ–≥—А–∞–љ–Є—З–µ–љ–Є–є**: —Б–Є—Б—В–µ–Љ–∞ —Б–Њ–Њ–±—Й–∞–µ—В –њ—А–Є–ї–Њ–ґ–µ–љ–Є—О, –Ї–∞–Ї–Є–µ –Њ–±–ї–∞—Б—В–Є —Н–Ї—А–∞–љ–∞ –Ј–∞–љ—П—В—Л —Б–Є—Б—В–µ–Љ–љ—Л–Љ–Є —Н–ї–µ–Љ–µ–љ—В–∞–Љ–Є (status bar, navigation bar, IME, display cutout). –≠—В–Њ —А–µ–∞–ї–Є–Ј–∞—Ж–Є—П –њ—А–Є–љ—Ж–Є–њ–∞ **information hiding** (Parnas, 1972): –њ—А–Є–ї–Њ–ґ–µ–љ–Є–µ –љ–µ –Ј–љ–∞–µ—В –Њ –Ї–Њ–љ–Ї—А–µ—В–љ—Л—Е —Б–Є—Б—В–µ–Љ–љ—Л—Е —Н–ї–µ–Љ–µ–љ—В–∞—Е вАФ —В–Њ–ї—М–Ї–Њ –Њ–± –∞–±—Б—В—А–∞–Ї—В–љ—Л—Е –Њ–≥—А–∞–љ–Є—З–µ–љ–Є—П—Е.

### –Ъ–Њ–і: –љ–∞–≤–Є–≥–∞—Ж–Є—П –њ–Њ View hierarchy

```kotlin
class DebugActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // –Я–Њ—Б–ї–µ setContentView –Љ–Њ–ґ–љ–Њ –Є—Б—Б–ї–µ–і–Њ–≤–∞—В—М –Є–µ—А–∞—А—Е–Є—О
        window.decorView.post {
            printViewHierarchy()
        }
    }

    private fun printViewHierarchy() {
        // 1. –Я–Њ–ї—Г—З–Є—В—М DecorView
        val decorView = window.decorView
        Log.d("Window", "DecorView: $decorView")
        Log.d("Window", "DecorView parent: ${decorView.parent}")
        // parent = ViewRootImpl (–љ–µ View!)

        // 2. –Я–Њ–ї—Г—З–Є—В—М content container
        val contentParent = findViewById<ViewGroup>(android.R.id.content)
        Log.d("Window", "Content parent: $contentParent")
        Log.d("Window", "Content children: ${contentParent.childCount}")

        // 3. –Я–Њ–ї—Г—З–Є—В—М –≤–∞—И –Ї–Њ—А–љ–µ–≤–Њ–є View
        val myRootView = contentParent.getChildAt(0)
        Log.d("Window", "My root view: $myRootView")

        // 4. Traverse –≤—Б—С –і–µ—А–µ–≤–Њ
        fun traverse(view: View, depth: Int = 0) {
            val indent = "  ".repeat(depth)
            val id = try {
                resources.getResourceEntryName(view.id)
            } catch (e: Exception) {
                "no-id"
            }
            Log.d("Window", "$indent${view.javaClass.simpleName} (id=$id)")
            if (view is ViewGroup) {
                for (i in 0 until view.childCount) {
                    traverse(view.getChildAt(i), depth + 1)
                }
            }
```

### [[android-activity-lifecycle]]
Window lifecycle —В–µ—Б–љ–Њ –њ—А–Є–≤—П–Ј–∞–љ –Ї Activity lifecycle: Window —Б–Њ–Ј–і–∞—С—В—Б—П –≤ Activity.attach() (–і–Њ onCreate), DecorView –і–Њ–±–∞–≤–ї—П–µ—В—Б—П –≤ WindowManager –≤ makeVisible() (–њ–Њ—Б–ї–µ onResume), –∞ –њ—А–Є finish() Window —Г–і–∞–ї—П–µ—В—Б—П –Є–Ј WMS. –Я–Њ–љ–Є–Љ–∞–љ–Є–µ —Н—В–Њ–є –њ–Њ—Б–ї–µ–і–Њ–≤–∞—В–µ–ї—М–љ–Њ—Б—В–Є –Њ–±—К—П—Б–љ—П–µ—В, –њ–Њ—З–µ–Љ—Г UI –љ–µ –≤–Є–і–µ–љ –≤ onCreate(), –њ–Њ—З–µ–Љ—Г Dialog crash –њ—А–Є –Є—Б–њ–Њ–ї—М–Ј–Њ–≤–∞–љ–Є–Є Application Context (–љ–µ—В window token), –Є –Ї–Њ–≥–і–∞ –±–µ–Ј–Њ–њ–∞—Б–љ–Њ –њ–Њ–Ї–∞–Ј—Л–≤–∞—В—М PopupWindow. –Ш–Ј—Г—З–Є—В–µ Activity lifecycle –њ–µ—А–≤—Л–Љ.
