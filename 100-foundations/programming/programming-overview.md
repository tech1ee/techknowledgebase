# Programming Foundations: обзор раздела

> Принципы, паттерны, тестирование, обработка ошибок, парадигмы и инструменты — фундамент качественного кода

## Как использовать этот раздел

Принципы и паттерны — это **эвристики**, не законы физики. Они помогают в большинстве случаев, но слепое следование приводит к overengineering.

```
Правильный подход:
  Понять ПОЧЕМУ принцип работает
    → Оценить контекст: размер проекта, команда, сроки
      → Применить с учётом trade-offs

НЕ: "SOLID говорит — значит делаю"
ДА: "SOLID здесь поможет, потому что..."
```
