# Fast & Slow Pointers Pattern

## Теоретические основы

> **Алгоритм Флойда обнаружения цикла (1967)** — метод с двумя указателями («черепаха и заяц»), доказывающий наличие цикла в последовательности итерируемых значений за `O(n)` времени и `O(1)` памяти.

### Формальное доказательство (sketch)

Пусть последовательность содержит цикл длины `C` с расстоянием `μ` от начала до входа в цикл. Slow проходит `μ + k` шагов, Fast — `2(μ + k)` шагов. В момент встречи оба внутри цикла: `2(μ + k) - (μ + k) = μ + k ≡ 0 (mod C)`. Следовательно, встреча гарантирована не позднее чем через `μ + C` шагов.

### Нахождение начала цикла

После встречи: сбросить один указатель на начало, оба двигать по одному шагу. Встретятся ровно на входе в цикл (позиция `μ`), т.к. расстояние от точки встречи до входа = расстоянию от начала до входа (mod C).

### Применения паттерна

| Задача | Суть | Сложность |
|--------|------|-----------|
| Обнаружение цикла | Fast и Slow встретятся ↔ цикл существует | `O(n)` / `O(1)` |
| Середина списка | Когда Fast дойдёт до конца, Slow — на середине | `O(n)` / `O(1)` |
| Happy Number | Последовательность сумм квадратов цифр — цикл или 1 | `O(log n)` / `O(1)` |
| Palindrome Linked List | Найти середину → развернуть вторую половину → сравнить | `O(n)` / `O(1)` |

Паттерн является специализацией [[two-pointers-pattern]] (Same Direction, разная скорость) — см. также [[linked-lists]].

## TL;DR

Fast & Slow Pointers (алгоритм Флойда, "Черепаха и Заяц") — паттерн с двумя указателями, движущимися с **разной скоростью**. Slow движется на 1 шаг, Fast на 2 шага. **Ключевое свойство:** если есть цикл — они обязательно встретятся; если нет — Fast достигнет конца. Используется для: обнаружения циклов, поиска середины списка, определения начала цикла, Happy Number. **Сложность: O(n) время, O(1) память** — эффективнее HashSet.

## Часть 1: Интуиция без кода

> **Цель:** понять ИДЕЮ Fast & Slow Pointers до любого кода. Если понимаешь эти аналогии — паттерн уже понятен.

### Ты уже знаешь этот паттерн

Этот паттерн встречается в повседневной жизни:

| Ситуация | Как это работает | Паттерн |
|----------|------------------|---------|
| **Два бегуна на круговом стадионе** | Быстрый обгонит медленного и догонит его сзади | Cycle Detection |
| **Часовая и минутная стрелки** | Минутная стрелка "догоняет" часовую каждый час | Cycle Detection |
| **Поиск середины верёвки** | Складываешь пополам, держа оба конца | Find Middle |
| **Гонки кроликов** | Два кролика: один прыгает на 1 метр, другой на 2 | Floyd's Algorithm |

### Аналогия 1: Беговая дорожка

Представь круговую беговую дорожку длиной 400 метров:

```
        ┌──────────────────────────┐
       /                            \
      │    🐢 Черепаха               │
      │    (1 м/сек)                 │
      │                              │
      │              🐇 Заяц         │
       \           (2 м/сек)        /
        └──────────────────────────┘
                  СТАРТ

Оба стартуют с одной точки.
- Черепаха: 1 метр в секунду
- Заяц: 2 метра в секунду

Через 400 секунд:
- Черепаха пробежала 400 м (1 круг)
- Заяц пробежал 800 м (2 круга)

Заяц ДОГНАЛ черепаху на круге! Они встретились.
```

**А что если дорожка прямая (без цикла)?**

```
СТАРТ ──────────────────────────────────────> ФИНИШ

🐢 → → → → → →
🐇 → → → → → → → → → → → → (дошёл до финиша первым)

Заяц дойдёт до конца. Встречи НЕ БУДЕТ.
```

**Вывод:** Если встретились — был цикл. Если заяц достиг конца — цикла нет.

### Аналогия 2: Часовые стрелки

```
    12
   /  \
  9    3   ← Минутная стрелка (fast)
   \  /    ← Часовая стрелка (slow)
    6

Минутная: 360° за 60 минут = 6° в минуту
Часовая:  360° за 12 часов = 0.5° в минуту

Разница скоростей: 6 - 0.5 = 5.5° в минуту

Минутная догоняет часовую каждые 360/5.5 ≈ 65 минут.
Это и есть "встреча" в цикле!
```

### Аналогия 3: Поиск середины

Нужно найти середину верёвки без измерения:

```
Способ 1 (два прохода):
  1. Пройди всю верёвку, посчитай длину
  2. Вернись на половину
  → 2 прохода, O(2n)

Способ 2 (Fast/Slow):
  1. Два пальца на начале
  2. Один палец двигается в 2 раза быстрее
  3. Когда быстрый дойдёт до конца — медленный в середине!
  → 1 проход, O(n)

Верёвка: ════════════════════════════════
         ↑                    ↑
        slow                fast (2x скорость)

         ════════════════════════════════
                    ↑                    ↑
                   slow                fast → достиг конца!
                    ↑
                СЕРЕДИНА!
```

### Главный инсайт: Почему они ОБЯЗАТЕЛЬНО встретятся?

```
┌─────────────────────────────────────────────────────────────────┐
│                    МАТЕМАТИЧЕСКОЕ ОБОСНОВАНИЕ                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Пусть оба в цикле длиной C.                                   │
│  Позиция slow = S, позиция fast = F                            │
│                                                                 │
│  После одного шага:                                            │
│    slow' = S + 1                                                │
│    fast' = F + 2                                                │
│                                                                 │
│  Разница: (F + 2) - (S + 1) = F - S + 1                        │
│                                                                 │
│  То есть разница УВЕЛИЧИВАЕТСЯ на 1 каждый шаг!               │
│  Эквивалентно: fast ПРИБЛИЖАЕТСЯ к slow на 1 шаг за итерацию.  │
│                                                                 │
│  Почему? Представь: slow стоит, fast идёт навстречу            │
│  со скоростью 1 (относительная скорость = 2 - 1 = 1).          │
│                                                                 │
│  Максимальное расстояние в цикле = C - 1                       │
│  Значит, максимум через C - 1 шагов они встретятся.            │
│                                                                 │
│  Общая сложность: O(n), где n = длина "хвоста" + длина цикла   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Часть 2: Почему Fast & Slow бывает сложен?

> **Цель:** понять типичные трудности и как их избежать.

### Отличие от Two Pointers

**Главная путаница:** Fast & Slow — это ПОДВИД Two Pointers, но с ДРУГОЙ логикой:

| Критерий | Two Pointers (Opposite) | Fast & Slow |
|----------|------------------------|-------------|
| **Движение** | Навстречу друг другу | В одном направлении |
| **Скорость** | Одинаковая | Разная (1 vs 2) |
| **Данные** | Отсортированные массивы | Связные списки, циклы |
| **Цель** | Поиск пары, сужение диапазона | Обнаружение цикла, середина |
| **Инвариант** | left < right | fast != null |

### Типичные трудности

#### 1. Почему проверять fast.next?

```kotlin
// ОШИБКА — NullPointerException:
while (fast != null) {
    fast = fast.next.next  // Если fast.next == null, CRASH!
}

// ПРАВИЛЬНО:
while (fast != null && fast.next != null) {
    fast = fast.next.next  // Безопасно
}

ПОЧЕМУ? Заяц делает ДВА прыжка:
  fast → fast.next → fast.next.next
         ↑
         Этот может быть null!

Если fast.next == null, то fast.next.next вызовет NPE.
```

#### 2. Когда использовать fast != null vs fast.next != null?

```
Для ЧЁТНОГО количества узлов (2, 4, 6...):
  1 → 2 → 3 → 4 → null
          ↑
         slow (после while fast != null && fast.next != null)

Для НЕЧЁТНОГО количества узлов (1, 3, 5...):
  1 → 2 → 3 → null
      ↑
     slow

ПРАВИЛО: fast != null && fast.next != null
  → slow будет на ПЕРВОЙ середине (для чётного списка)
  → slow будет точно посередине (для нечётного)
```

#### 3. Как найти НАЧАЛО цикла?

```
После обнаружения цикла (slow == fast):

АЛГОРИТМ:
1. Сбрось slow на head
2. Оставь fast на точке встречи
3. Оба двигаются по 1 шагу
4. Где встретятся — там начало цикла!

ПОЧЕМУ ЭТО РАБОТАЕТ?

Пусть:
- L = расстояние от head до начала цикла
- C = длина цикла
- K = расстояние от начала цикла до точки встречи

Когда они встретились:
- slow прошёл: L + K
- fast прошёл: L + K + nC (n полных кругов)

Так как fast = 2 × slow:
  L + K + nC = 2(L + K)
  nC = L + K
  L = nC - K

Это значит: расстояние от head до начала цикла (L)
равно расстоянию от точки встречи до начала цикла (nC - K).

Поэтому если один идёт от head, другой от точки встречи —
они встретятся на начале цикла!
```

#### 4. Happy Number — где тут цикл?

```
Happy Number: повторяй сумму квадратов цифр, пока не 1.

19 → 1² + 9² = 82
82 → 8² + 2² = 68
68 → 6² + 8² = 100
100 → 1² + 0² + 0² = 1 ✓ HAPPY!

2 → 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4 (ЦИКЛ!)

ГДЕ ЦИКЛ? Последовательность чисел!
Если не приходит к 1 — попадает в бесконечный цикл.

Применяем Fast/Slow к ЧИСЛАМ:
- slow = getNext(slow)
- fast = getNext(getNext(fast))

Если встретились и значение != 1 → не happy
Если slow == 1 → happy!
```

### Что отличает новичка от эксперта

| Новичок | Эксперт |
|---------|---------|
| Забывает проверить `fast.next` | Всегда пишет `fast != null && fast.next != null` |
| Не понимает математику за "найти начало цикла" | Знает доказательство L = nC - K |
| Использует HashSet для cycle detection | Знает, что Fast/Slow экономит O(n) памяти |
| Путает с Sliding Window | Понимает: разная скорость vs разный размер окна |

## Часть 3: Ментальные модели для Fast & Slow

> **Цель:** дать 3 разных способа ДУМАТЬ о паттерне. Выбери тот, который ближе.

### Модель 1: "Догонялки на круге"

**Идея:** Два объекта на замкнутой траектории. Быстрый обязательно догонит медленного.

```
┌─────────────────────────────────────────────────────────────────┐
│                        ДОГОНЯЛКИ НА КРУГЕ                       │
│                                                                 │
│         ┌───────────────────┐                                  │
│        /                     \                                 │
│       │    🐢                 │                                │
│       │    slow               │  Цикл длиной C                 │
│       │                       │                                │
│       │          🐇           │                                │
│        \        fast         /                                 │
│         └───────────────────┘                                  │
│                                                                 │
│   Каждую итерацию:                                             │
│   - slow делает 1 шаг                                          │
│   - fast делает 2 шага                                         │
│   - расстояние между ними сокращается на 1                     │
│                                                                 │
│   Максимум через C шагов — встреча!                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Когда использовать:** Cycle Detection в списках, Happy Number.

### Модель 2: "Двойная скорость для середины"

**Идея:** Если один пройдёт весь путь, другой — половину.

```
┌─────────────────────────────────────────────────────────────────┐
│                     ДВОЙНАЯ СКОРОСТЬ                            │
│                                                                 │
│   Путь:  ═══════════════════════════════════════                │
│          ↑                    ↑                 ↑               │
│         START               slow              fast              │
│                          (n/2 шагов)       (n шагов)            │
│                                                                 │
│   Когда fast дошёл до конца:                                   │
│   - fast прошёл n шагов                                         │
│   - slow прошёл n/2 шагов = СЕРЕДИНА!                          │
│                                                                 │
│   Почему? slow = fast / 2                                       │
│           Если fast = n, то slow = n/2                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Когда использовать:** Find Middle of Linked List, Palindrome Linked List.

### Модель 3: "Относительное движение"

**Идея:** С точки зрения slow, fast приближается со скоростью 1.

```
┌─────────────────────────────────────────────────────────────────┐
│                    ОТНОСИТЕЛЬНОЕ ДВИЖЕНИЕ                       │
│                                                                 │
│   Реальная картина:                                            │
│     slow:  ● → ● → ● → ● → ...                                 │
│     fast:  ● → → ● → → ● → → ● → ...                           │
│                                                                 │
│   С точки зрения slow (она "стоит"):                           │
│     slow:  ●     ●     ●     ●                                 │
│     fast:  ● → ● → ● → ● (приближается на 1 за шаг)            │
│                                                                 │
│   Относительная скорость = 2 - 1 = 1                           │
│   Fast приближается к slow на 1 позицию за итерацию.           │
│                                                                 │
│   В цикле длиной C:                                            │
│   Максимальное расстояние = C - 1                               │
│   Время до встречи = C - 1 шагов                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Когда использовать:** Понимание математики, доказательства корректности.

## Что это такое?

### Объяснение для 5-летнего

Представь длинную цепочку вагонов поезда. Тебе нужно узнать, замкнута ли она в круг (последний вагон соединён с первым).

У тебя два робота: Черепашка и Зайчик.
- Черепашка прыгает на 1 вагон за шаг
- Зайчик прыгает на 2 вагона за шаг

Если поезд — **прямая линия**: Зайчик дойдёт до конца первым. Конец!

Если поезд — **круг**: Зайчик будет бегать по кругу и рано или поздно догонит Черепашку!

```
Прямой поезд:
  🐢 → → → → 🏁
  🐇 → → → → → → → → 🏁 (пришёл первым)

Круговой поезд:
    ┌──→──┐
    │ 🐇  │
    ↑     ↓
    │ 🐢  │
    └──←──┘
  Зайчик догнал Черепашку!
```

### Формальное определение

**Fast & Slow Pointers (алгоритм Флойда)** — алгоритмический паттерн, использующий два указателя с разными скоростями перемещения по структуре данных для обнаружения циклов или нахождения определённых позиций.

**Ключевые свойства:**
- **Два указателя:** slow (скорость 1), fast (скорость 2)
- **Одно направление:** оба движутся "вперёд"
- **Инвариант:** если есть цикл — встретятся; если нет — fast достигнет null
- **Константная память:** O(1) дополнительной памяти

**Основные применения:**

```
1. Cycle Detection (обнаружение цикла)
   Вход: связный список
   Выход: true/false — есть ли цикл

2. Find Cycle Start (начало цикла)
   Вход: связный список с циклом
   Выход: узел, с которого начинается цикл

3. Find Middle (середина списка)
   Вход: связный список
   Выход: средний узел

4. Happy Number (счастливое число)
   Вход: число
   Выход: приводит ли к 1 или зацикливается
```

## Терминология

| Термин | Определение | Пример |
|--------|-------------|--------|
| **Slow Pointer** | Указатель со скоростью 1 шаг | `slow = slow.next` |
| **Fast Pointer** | Указатель со скоростью 2 шага | `fast = fast.next.next` |
| **Cycle** | Замкнутая последовательность узлов | Последний узел указывает на предыдущий |
| **Cycle Length** | Количество узлов в цикле | Если цикл 3→4→5→3, длина = 3 |
| **Tail** | Часть списка до входа в цикл | Узлы перед началом цикла |
| **Meeting Point** | Точка встречи slow и fast | Узел, где slow == fast |
| **Cycle Start** | Узел, с которого начинается цикл | Первый узел цикла |
| **Floyd's Algorithm** | Алгоритм черепахи и зайца | Другое название Fast/Slow |

## Как это работает?

### Задача 1: Cycle Detection (Обнаружение цикла)

```
Linked List Cycle — есть ли цикл в списке?

Список БЕЗ цикла:
    1 → 2 → 3 → 4 → null

Список С циклом:
    1 → 2 → 3 → 4
            ↑   ↓
            6 ← 5

Алгоритм:
    slow = head, fast = head

    while (fast != null && fast.next != null):
        slow = slow.next          // +1
        fast = fast.next.next     // +2

        if slow == fast:
            return true  // Цикл найден!

    return false  // fast достиг null — цикла нет
```

**Пошаговая симуляция (с циклом):**

```
Список: 1 → 2 → 3 → 4 → 5 → 3 (5 указывает обратно на 3)

Шаг 0: slow = 1, fast = 1
Шаг 1: slow = 2, fast = 3
Шаг 2: slow = 3, fast = 5
Шаг 3: slow = 4, fast = 4  ← ВСТРЕТИЛИСЬ! Цикл обнаружен.

Визуализация:
    Step 0:  [S,F] → 2 → 3 → 4 → 5 ─┐
                         ↑──────────┘

    Step 1:  1 → [S] → [F] → 4 → 5 ─┐
                         ↑──────────┘

    Step 2:  1 → 2 → [S] → 4 → [F] ─┐
                         ↑──────────┘

    Step 3:  1 → 2 → 3 → [S,F] → 5 ─┐  ← Встреча!
                         ↑──────────┘
```

### Задача 2: Find Cycle Start (Найти начало цикла)

```
Дано: список с циклом
Найти: узел, с которого начинается цикл

    1 → 2 → 3 → 4 → 5
            ↑       ↓
            └───────┘

    Ответ: узел 3 (начало цикла)

Алгоритм:
    1. Найди точку встречи slow и fast (cycle detection)
    2. Сбрось slow на head
    3. Двигай оба по 1 шагу
    4. Где встретятся — там начало цикла

Почему это работает?

    Пусть:
    - L = расстояние от head до начала цикла
    - C = длина цикла
    - K = расстояние от начала цикла до точки встречи

    При первой встрече:
    - slow прошёл: L + K
    - fast прошёл: L + K + nC (n полных кругов)

    Так как fast = 2 × slow:
        L + K + nC = 2(L + K)
        nC = L + K
        L = nC - K

    Это означает:
    L = расстояние от head до начала цикла
    nC - K = расстояние от точки встречи до начала цикла (через n-1 круг)

    Они РАВНЫ! Поэтому один с head, другой с meeting point —
    встретятся ровно на начале цикла.
```

**Пошаговая симуляция:**

```
Список: 1 → 2 → 3 → 4 → 5 → 3 (L=2, C=3, K=1)

Фаза 1: Cycle Detection
    slow: 1 → 2 → 3 → 4
    fast: 1 → 3 → 5 → 4
    Встреча: узел 4

Фаза 2: Find Start
    Сбросить slow на head (узел 1)
    fast остаётся на узле 4

    Шаг 1: slow = 2, fast = 5
    Шаг 2: slow = 3, fast = 3  ← ВСТРЕЧА! Узел 3 — начало цикла.
```

### Задача 3: Find Middle (Середина списка)

```
Дано: связный список
Найти: средний узел

Нечётное количество:  1 → 2 → 3 → 4 → 5
                              ↑
                           середина (3)

Чётное количество:    1 → 2 → 3 → 4
                          ↑
                       первая середина (2)
                       или вторая (3) — зависит от условия

Алгоритм:
    slow = head, fast = head

    while (fast != null && fast.next != null):
        slow = slow.next
        fast = fast.next.next

    return slow  // slow на середине!
```

**Пошаговая симуляция:**

```
Список: 1 → 2 → 3 → 4 → 5 (нечётный)

Шаг 0: slow = 1, fast = 1
Шаг 1: slow = 2, fast = 3
Шаг 2: slow = 3, fast = 5
       fast.next == null → СТОП!

Ответ: slow = 3 (середина)

Визуализация:
    1 → 2 → 3 → 4 → 5
    S   S   S          (slow: 3 шага)
    F       F       F  (fast: 4 шага, упёрся в null)
            ↑
         СЕРЕДИНА
```

### Задача 4: Happy Number

```
Определение: Число "счастливое", если сумма квадратов его цифр
             приводит к 1. Иначе зацикливается.

Пример 1: 19 (happy)
    19 → 1² + 9² = 82
    82 → 64 + 4 = 68
    68 → 36 + 64 = 100
    100 → 1 + 0 + 0 = 1 ✓

Пример 2: 2 (not happy)
    2 → 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4 → ... (цикл!)

Алгоритм:
    Применяем Fast/Slow к последовательности чисел!

    slow = n
    fast = n

    do:
        slow = sumOfSquares(slow)
        fast = sumOfSquares(sumOfSquares(fast))
    while (slow != fast)

    return slow == 1

Интуиция:
    - Если число happy → slow дойдёт до 1, fast тоже
    - Если not happy → образуется цикл, slow и fast встретятся НЕ на 1
```

|--------|-------|--------|------------|
| Cycle Detection | O(n) | O(1) | n = длина списка |
| Find Cycle Start | O(n) | O(1) | Две фазы, обе O(n) |
| Find Middle | O(n) | O(1) | Один проход |

| Подход | Время | Память | Преимущество |
|--------|-------|--------|--------------|
| HashSet | O(n) | O(n) | Проще понять |
| Fast/Slow | O(n) | O(1) | Экономия памяти! |

### Kotlin

```kotlin
// ═══════════════════════════════════════════════════════════════════════════
// CYCLE DETECTION — Обнаружение цикла
// ═══════════════════════════════════════════════════════════════════════════

/**
 * Проверяет, есть ли цикл в связном списке.
 *
 * ИДЕЯ: slow +1, fast +2. Если есть цикл — встретятся.
 *
 * ПРИМЕР: 1 → 2 → 3 → 4 → 2 (4 указывает на 2)
 *         slow: 1→2→3→4
 *         fast: 1→3→2→4
 *         Встреча на 4!
 */
fun hasCycle(head: ListNode?): Boolean {
    var slow = head
    var fast = head

    // fast.next проверяем, чтобы fast.next.next не вызвал NPE
    while (fast != null && fast.next != null) {
        slow = slow?.next           // +1 шаг
        fast = fast.next?.next      // +2 шага

        if (slow == fast) {
            return true  // Цикл найден!
        }
    }

    return false  // fast достиг null — цикла нет
}

// ═══════════════════════════════════════════════════════════════════════════
// FIND CYCLE START — Найти начало цикла
// ═══════════════════════════════════════════════════════════════════════════

/**
 * Находит узел, с которого начинается цикл.
 *
 * ИДЕЯ: После встречи сбрось slow на head, двигай оба по 1.
 *       Встретятся на начале цикла (математика: L = nC - K).
 *
 * ПРИМЕР: 1 → 2 → 3 → 4 → 5 → 3
 *         Встреча slow и fast на узле 4
 *         slow=1, fast=4
 *         slow: 1→2→3
 *         fast: 4→5→3
 *         Встреча на 3 — начало цикла!
 */
fun detectCycle(head: ListNode?): ListNode? {
    var slow = head
    var fast = head

    // Фаза 1: Найти точку встречи
    while (fast != null && fast.next != null) {
        slow = slow?.next
        fast = fast.next?.next

        if (slow == fast) {
            // Фаза 2: Найти начало цикла
            slow = head
            while (slow != fast) {
                slow = slow?.next
                fast = fast?.next  // Теперь оба по 1 шагу!
            }
            return slow  // Начало цикла
        }
    }

    return null  // Цикла нет
}

// ═══════════════════════════════════════════════════════════════════════════
// FIND MIDDLE — Найти середину списка
// ═══════════════════════════════════════════════════════════════════════════

/**
 * Находит средний узел связного списка.
 *
 * ИДЕЯ: fast проходит весь путь, slow — половину.
 *       Когда fast достиг конца, slow на середине.
 *
 * ПРИМЕР: 1 → 2 → 3 → 4 → 5
 *         slow: 1→2→3 (3 шага до середины)
 *         fast: 1→3→5 (достиг конца)
 *         Ответ: 3
 */
fun middleNode(head: ListNode?): ListNode? {
    var slow = head
    var fast = head

    while (fast != null && fast.next != null) {
        slow = slow?.next
        fast = fast.next?.next
    }

    return slow  // slow на середине
}

// ═══════════════════════════════════════════════════════════════════════════
// HAPPY NUMBER — Счастливое число
// ═══════════════════════════════════════════════════════════════════════════

/**
 * Проверяет, является ли число "счастливым".
 *
 * ИДЕЯ: Последовательность сумм квадратов цифр — либо 1, либо цикл.
 *       Применяем Fast/Slow к числам.
 *
 * ПРИМЕР: 19 → 82 → 68 → 100 → 1 ✓ (happy)
 *         2 → 4 → 16 → ... → 4 (цикл, not happy)
 */
fun isHappy(n: Int): Boolean {
    var slow = n
    var fast = n

    do {
        slow = sumOfSquares(slow)
        fast = sumOfSquares(sumOfSquares(fast))
    } while (slow != fast)

    return slow == 1
}

private fun sumOfSquares(n: Int): Int {
    var num = n
    var sum = 0
    while (num > 0) {
        val digit = num % 10
        sum += digit * digit
        num /= 10
    }
    return sum
}

// ═══════════════════════════════════════════════════════════════════════════
// PALINDROME LINKED LIST — Палиндром в списке
// ═══════════════════════════════════════════════════════════════════════════

/**
 * Проверяет, является ли связный список палиндромом.
 *
 * ИДЕЯ: 1) Найти середину (Fast/Slow)
 *       2) Развернуть вторую половину
 *       3) Сравнить первую и вторую половины
 *
 * ПРИМЕР: 1 → 2 → 2 → 1
 *         Середина: 2 (первая)
 *         Вторая половина: 2 → 1
 *         Развернуть: 1 → 2
 *         Сравнить: 1→2 vs 1→2 ✓
 */
fun isPalindrome(head: ListNode?): Boolean {
    if (head?.next == null) return true

    // 1. Найти середину
    var slow = head
    var fast = head
    while (fast?.next != null && fast.next?.next != null) {
        slow = slow?.next
        fast = fast.next?.next
    }

    // 2. Развернуть вторую половину
    var secondHalf = reverseList(slow?.next)

    // 3. Сравнить
    var firstHalf = head
    while (secondHalf != null) {
        if (firstHalf?.`val` != secondHalf.`val`) {
            return false
        }
        firstHalf = firstHalf.next
        secondHalf = secondHalf.next
    }

    return true
}

private fun reverseList(head: ListNode?): ListNode? {
    var prev: ListNode? = null
    var curr = head
    while (curr != null) {
        val next = curr.next
        curr.next = prev
        prev = curr
        curr = next
    }
    return prev
}
```

```python
# ═══════════════════════════════════════════════════════════════════════════
# CYCLE DETECTION — Обнаружение цикла
# ═══════════════════════════════════════════════════════════════════════════

def hasCycle(head: ListNode) -> bool:
    """
    Проверяет, есть ли цикл в связном списке.

    Время: O(n), Память: O(1)
    """
    slow = fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

        if slow == fast:
            return True

    return False

# FIND CYCLE START — Найти начало цикла
# ═══════════════════════════════════════════════════════════════════════════

def detectCycle(head: ListNode) -> ListNode:
    """
    Находит узел, с которого начинается цикл.

    Время: O(n), Память: O(1)
    """
    slow = fast = head

    # Фаза 1: Найти точку встречи
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

        if slow == fast:
            # Фаза 2: Найти начало цикла
            slow = head
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow

    return None


# ═══════════════════════════════════════════════════════════════════════════
# FIND MIDDLE — Найти середину
# ═══════════════════════════════════════════════════════════════════════════

def middleNode(head: ListNode) -> ListNode:
    """
    Возвращает средний узел списка.
    Для чётного количества — второй средний.

    Время: O(n), Память: O(1)
    """
    slow = fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    return slow


# ═══════════════════════════════════════════════════════════════════════════
# HAPPY NUMBER — Счастливое число
# ═══════════════════════════════════════════════════════════════════════════

def isHappy(n: int) -> bool:
    """
    Проверяет, является ли число счастливым.

    Счастливое число: сумма квадратов цифр приводит к 1.
    """
    def sum_of_squares(num: int) -> int:
        total = 0
        while num > 0:
            digit = num % 10
            total += digit * digit
            num //= 10
        return total

    slow = fast = n

    while True:
        slow = sum_of_squares(slow)
        fast = sum_of_squares(sum_of_squares(fast))

        if slow == fast:
            break

    return slow == 1


# ═══════════════════════════════════════════════════════════════════════════
# FIND DUPLICATE — Найти дубликат (задача без модификации массива)
# ═══════════════════════════════════════════════════════════════════════════

def findDuplicate(nums: list[int]) -> int:
    """
    Находит дубликат в массиве [1, n] с n+1 элементами.

    ИДЕЯ: Массив как связный список!
          nums[i] = следующий индекс
          Дубликат = начало цикла

    Пример: [1, 3, 4, 2, 2]
            0 → 1 → 3 → 2 → 4 → 2 (цикл на 2!)

    Время: O(n), Память: O(1)
    """
    # Фаза 1: Найти точку встречи
    slow = fast = nums[0]

    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast:
            break

    # Фаза 2: Найти начало цикла
    slow = nums[0]
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]

    return slow
```

## Когда применять Fast & Slow?

### Сигналы для распознавания

```
Используй Fast & Slow, если:

✅ "Есть ли цикл?" (linked list, последовательность чисел)
✅ "Найти середину" (без подсчёта длины)
✅ "Найти n-й элемент с конца" (вариация: gap pointers)
✅ "Happy Number / последовательность зацикливается?"
✅ "Найти дубликат в массиве [1..n] за O(1) памяти"
✅ "Палиндром в связном списке"

НЕ используй Fast & Slow, если:

❌ Нужен random access (массивы — используй Two Pointers)
❌ Отсортированные данные (используй Binary Search или Two Pointers)
❌ Sliding Window задачи (другой паттерн)
```

### Шаблон решения

```
1. ОПРЕДЕЛИ, где цикл:
   - Связный список? → указатели .next
   - Последовательность чисел? → f(n) как "next"
   - Массив как "linked list"? → nums[i] как "next"

2. ИНИЦИАЛИЗИРУЙ:
   slow = start
   fast = start

3. ДВИГАЙ:
   while (условие выхода):
       slow = next(slow)
       fast = next(next(fast))

4. ОБРАБОТАЙ результат:
   - Если встретились → есть цикл
   - Если fast == null → нет цикла
   - Нужно начало цикла? → Фаза 2 (reset slow to head)
```

## Частые ошибки и как их избежать

### Ошибка 1: Забыть проверку fast.next

```kotlin
// НЕПРАВИЛЬНО:
while (fast != null) {
    fast = fast.next.next  // NPE если fast.next == null!
}

// ПРАВИЛЬНО:
while (fast != null && fast.next != null) {
    fast = fast.next.next
}
```

### Ошибка 2: Неправильная инициализация для Find Cycle Start

```kotlin
// НЕПРАВИЛЬНО:
fun findCycleStart(head: ListNode?): ListNode? {
    var slow = head
    var fast = head?.next  // ОШИБКА! Они должны стартовать с одного места
    ...
}

// ПРАВИЛЬНО:
fun findCycleStart(head: ListNode?): ListNode? {
    var slow = head
    var fast = head  // Оба с head
    ...
}
```

### Ошибка 3: Перепутать do-while и while для Happy Number

```kotlin
// НЕПРАВИЛЬНО:
while (slow != fast) {  // Сразу true, цикл не выполнится!
    slow = sumOfSquares(slow)
    fast = sumOfSquares(sumOfSquares(fast))
}

// ПРАВИЛЬНО:
do {
    slow = sumOfSquares(slow)
    fast = sumOfSquares(sumOfSquares(fast))
} while (slow != fast)
```

## Мифы и реальность

### Миф 1: "Fast/Slow работает только для linked lists"

**Реальность:** Паттерн работает для любой последовательности, где есть понятие "следующего элемента":
- Связные списки (node.next)
- Числовые последовательности (f(n))
- Массивы как implicit linked lists (nums[i])

### Миф 2: "HashSet проще и так же эффективен"

**Реальность:**
- HashSet: O(n) время, O(n) память
- Fast/Slow: O(n) время, O(1) память

Для списка из 10 миллионов узлов HashSet требует ~400MB памяти, Fast/Slow — 16 байт.

### Миф 3: "Математика за Find Cycle Start слишком сложная"

**Реальность:** Запомни одну формулу: **L = nC - K**

Где L = расстояние до цикла, C = длина цикла, K = расстояние от начала цикла до точки встречи.

Это означает: от head и от meeting point до начала цикла — одинаковое количество шагов.

### Миф 4: "Slow и Fast должны стартовать с разных позиций"

**Реальность:** Оба ОБЯЗАТЕЛЬНО стартуют с одной позиции (head).
Иначе математика "найти начало цикла" не сработает.

```
│  • Проверка: fast != null && fast.next != null                  │
│                                                                 │
│  ПОЧЕМУ O(1) память:                                            │
│  • Только 2 указателя вместо HashSet размера n                  │
│                                                                 │
│  КЛЮЧЕВАЯ МАТЕМАТИКА:                                           │
│  • Если есть цикл — slow и fast ОБЯЗАТЕЛЬНО встретятся          │
│  • Для начала цикла: L = nC - K                                 │
│                                                                 │
│  ПРОВЕРКА ПОНИМАНИЯ:                                            │
```

## Связь с другими темами

### [[two-pointers-pattern]]
Fast & Slow Pointers — это специализация общего паттерна Two Pointers, где указатели движутся с разной скоростью вместо навстречу друг другу или в одном направлении с одинаковой скоростью. Two Pointers формирует базовую интуицию работы с двумя переменными-указателями, и Fast/Slow расширяет эту идею для задач обнаружения циклов. Понимание Two Pointers (сходящиеся, расходящиеся, sliding window) помогает увидеть Fast/Slow как частный случай общего принципа. Рекомендуется освоить Two Pointers до изучения Fast/Slow.

### [[linked-lists]]
Связные списки — основная структура данных для применения Fast & Slow Pointers. Обнаружение циклов, поиск середины списка, определение начала цикла — все классические задачи паттерна работают именно с linked lists. Понимание устройства узлов (node.val, node.next), операций traversal и особенностей указателей в связных списках критически важно для корректной реализации. Без знания linked lists невозможно понять, почему `fast = fast.next.next` может вызвать NullPointerException и зачем нужна проверка `fast.next != null`.

### [[hash-tables]]
Hash-таблицы решают те же задачи, что и Fast/Slow Pointers (обнаружение циклов, поиск дубликатов), но за счёт O(n) дополнительной памяти. HashSet хранит все посещённые элементы и проверяет наличие за O(1), тогда как Fast/Slow достигает того же за O(1) памяти. Сравнение подходов учит выбирать между простотой реализации (HashSet) и оптимальностью по памяти (Floyd's algorithm). На интервью полезно предложить оба решения и объяснить tradeoff.

## Проверь себя

> [!question]- Почему быстрый указатель обязательно догонит медленного в цикле, и за сколько шагов?
> На каждом шаге разница расстояний между fast и slow уменьшается на 1 (fast +2, slow +1). Если длина цикла C, то встреча произойдёт максимум за C шагов после входа slow в цикл. Доказательство: расстояние d уменьшается d, d-1, d-2, ..., 0. Работает для любой длины цикла.

> [!question]- Как найти начало цикла в linked list (фаза 2 алгоритма Флойда)?
> После встречи fast и slow: ставим один указатель на head, другой оставляем в точке встречи. Оба двигаются по 1 шагу. Точка следующей встречи = начало цикла. Доказательство: если расстояние от head до начала цикла = a, от начала цикла до точки встречи = b, длина цикла = C, то a = C - b.

> [!question]- Задача Happy Number: почему можно использовать Fast/Slow вместо HashSet?
> Последовательность sum-of-squares-of-digits либо достигает 1 (happy), либо зацикливается. Это эквивалентно cycle detection в implicit linked list. Fast/Slow: O(1) память вместо O(n) для HashSet. Slow вычисляет один шаг, fast два. Если встретились и значение не 1 — not happy.

## Ключевые карточки

Что такое алгоритм Флойда (Tortoise and Hare)?
?
Два указателя: slow (шаг 1) и fast (шаг 2). Если цикл есть — встретятся. Если нет — fast достигнет null. Фаза 1: детекция цикла. Фаза 2: нахождение начала цикла (reset один к head, оба по 1). O(n) время, O(1) память.

Как найти середину linked list?
?
Slow шагает на 1, fast на 2. Когда fast.next == null или fast.next.next == null, slow на середине. Для чётной длины: slow на первом из двух средних (или втором, в зависимости от условия остановки). Используется в Merge Sort списков.

Какие задачи решает Fast/Slow кроме cycle detection?
?
1) Find Middle of Linked List. 2) Palindrome Linked List (middle + reverse + compare). 3) Reorder List (middle + reverse + merge). 4) Happy Number. 5) Circular Array Loop. 6) Find Duplicate Number (linked list reduction).

Чем Fast/Slow отличается от обычных Two Pointers?
?
Two Pointers: разные позиции, одинаковая скорость (или opposite direction). Fast/Slow: одинаковая начальная позиция, разная скорость (1 vs 2). Ключевое свойство: разная скорость создаёт 'относительное движение' для детекции циклов.

Как решить Find the Duplicate Number без модификации массива?
?
Массив [1,n] с одним дубликатом — implicit linked list: index -> value = next pointer. Дубликат = начало цикла. Floyd's: slow = nums[slow], fast = nums[nums[fast]]. Фаза 2 для начала цикла. O(n) время, O(1) память.
