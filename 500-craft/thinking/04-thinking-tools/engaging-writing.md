# Писать так, чтобы читали: контент в эпоху СДВГ

79% сканируют, не читают. 5% дочитывают до конца. Время фокуса упало с 2.5 минут (2004) до 47 секунд (2016). Главное сразу в первом абзаце. Короткие предложения. Визуальная структура. Открытые петли для любопытства.

---

## Проблема: почему тексты не дочитывают

### Статистика внимания

Исследования Gloria Mark (UC Irvine, изучает внимание с 2004 года):

```
Среднее время фокуса на одном экране:

2004:  ████████████████████████████████  2.5 минуты
2016:  ████████                          47 секунд

Падение: ~70% за 12 лет
```

Это не "люди стали тупее". Это адаптация к среде с бесконечным контентом.

### Как люди реально читают в интернете

Nielsen Norman Group (eye-tracking исследования):

```
F-паттерн чтения:

█████████████████████  ← Читают первую строку
█████████████████████
████████████           ← Сканируют начало строк
██████████
████████               ← Внимание падает
██████
████                   ← До конца доходят немногие
██
```

**79%** пользователей сканируют страницы
**16%** читают слово за словом
**~5%** дочитывают длинные тексты до конца

### Конкуренция за внимание

```
Количество контента в день (средний пользователь):

Emails:          ~120
Сообщения:       ~200
Посты в лентах:  ~300
Статьи/новости:  ~50+
                 ────────
                 ~670+ единиц контента

Время на каждую единицу: секунды
```

## Научная основа: почему техники работают

### Эффект Зейгарник (незавершённые задачи)

Блюма Зейгарник (1927) обнаружила: незавершённые задачи запоминаются лучше завершённых. Мозг "держит открытой вкладку" для незаконченного.

```
Официант помнит заказ     →    Заказ подан
пока он не выполнен            →    Забыл через минуту

Почему: мозг освобождает ресурсы после "закрытия"
```

**Нейробиологический механизм (теория поля Левина):**

```
Незавершённая задача создаёт "психологическое напряжение":

┌─────────────────────────────────────────────────────┐
│  Начало задачи                                      │
│        │                                            │
│        ▼                                            │
│  Активация нейронных цепей                          │
│  (создаётся "энергетическое поле")                  │
│        │                                            │
│        ├──► Задача завершена → Разрядка напряжения  │
│        │                       Память угасает       │
│        │                                            │
│        └──► Задача прервана → Напряжение сохраняется│
│                               Память остаётся       │
│                               активной              │
└─────────────────────────────────────────────────────┘

Это как открытая вкладка браузера: потребляет ресурсы,
пока её не закроешь.
```

**Важное уточнение (метаанализ 2025):**

Систематический обзор 2025 года показал: эффект **не универсален**. Он проявляется:
- В "расслабленных" условиях (низкий стресс, нет давления)
- При высокой личной вовлечённости в задачу
- Когда прерывание воспринимается как временное

**Когда эффект НЕ работает:**
- Высокий стресс или тревога (мозг занят выживанием)
- Принудительное прерывание (раздражение вместо интереса)
- Слишком много открытых задач одновременно ("Open Loop Fatigue")

**Применение к тексту:**
Открытая петля (незавершённая мысль) → Мозг хочет закрыть → Продолжает читать

Но только если читатель вовлечён и не перегружен другими "открытыми вкладками".

### Когнитивная нагрузка

```
Ресурсы внимания ограничены:

┌─────────────────────────────────────────┐
│ Доступная "оперативка" мозга: 100%      │
│                                         │
│ Простой текст:                          │
│ ███████████ Понимание контента (55%)    │
│ █████ Навигация по структуре (25%)      │
│ ████ Свободно (20%)                     │
│                                         │
│ Сложный текст (стена текста):           │
│ ██████████████████ Понимание (90%)      │
│ █████████ Навигация (45%)               │
│ [ПЕРЕГРУЗКА] → Закрыл вкладку           │
└─────────────────────────────────────────┘
```

**Вывод:** Чем проще навигация, тем больше ресурсов на понимание.

### Дофаминовая система

Короткие видео (TikTok, Reels) тренируют мозг на быстрые награды:

```
Исследование влияния коротких видео на внимание:

Потребление коротких видео    Способность к глубокому чтению
────────────────────────────  ─────────────────────────────
        ▲ Больше              Хуже ▼

Студенты с высоким потреблением коротких видео:
- Короче время внимания
- Ниже академическая успеваемость
- Сложнее концентрироваться на лекциях и чтении
```

**Что это значит для авторов:** Читатели приходят с "испорченными" ожиданиями. Нужно работать с этим, а не против этого.

## Техники: структура текста

### 1️⃣ Перевёрнутая пирамида (Inverted Pyramid)

Журналистская техника: главное — в начале, детали — потом.

```
Традиционный текст:           Перевёрнутая пирамида:

        /\                           ████████████
       /  \                          █ ГЛАВНОЕ  █
      /    \                         ████████████
     / Ввод \                             ▼
    /────────\                       ██████████
   / Контекст \                      █ Детали █
  /────────────\                     ██████████
 /   Развитие   \                        ▼
/────────────────\                    ████████
│   ВЫВОД/ГЛАВНОЕ │                   █ Фон █
└─────────────────┘                    ████████

Проблема: читатель                Преимущество: читатель
ушёл до вывода                    сразу получает ценность
```

**Принцип:** Можно остановиться в любой точке и унести главное.

**Как применять:**
- Первый абзац = ответ на вопрос "О чём это и зачем мне?"
- Каждый раздел начинается с вывода, не с предыстории
- TL;DR в начале, не в конце

### 2️⃣ Фронтлоудинг (Front-loading)

Не только статья, но каждый элемент начинается с главного:

```
❌ Плохо:                         ✅ Хорошо:
─────────────────                ─────────────────
"В ходе исследования,            "Исследование MIT: 83%
проведённого MIT в 2025           не помнят, что писали
году с участием 54                с ChatGPT."
студентов..."

Заголовок:                       Заголовок:
"Некоторые мысли о               "Почему ваш код —
разработке ПО"                    техдолг: 3 признака"

Первое предложение:              Первое предложение:
"Многие задаются                 "Переключение контекста
вопросом о                       стоит 23 минуты.
продуктивности..."               Вот как это исправить."
```

### 3️⃣ Визуальная иерархия

```
ЭЛЕМЕНТЫ СКАНИРОВАНИЯ (что видят при пролистывании):

1. Заголовки и подзаголовки     ← Первое, что видят
2. Жирный и курсивный текст     ← Второе
3. Списки и буллеты             ← Структура
4. Цифры и статистика           ← Конкретика
5. Картинки и схемы             ← Визуал
6. Первые слова абзацев         ← F-паттерн
7. Основной текст               ← Если зацепило

Правило: Если читать ТОЛЬКО элементы 1-6,
         смысл статьи должен быть понятен.
```

### 4️⃣ Чанкинг (разбиение на блоки)

```
❌ Стена текста:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod
tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim
veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea
commodo consequat. Duis aute irure dolor in reprehenderit in voluptate
velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat
cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id
est laborum...

✅ Разбито на блоки:

**Главная мысль в одном предложении.**

Короткий абзац с развитием (2-3 строки максимум).

- Буллет для перечисления
- Ещё один пункт
- И третий

> Цитата или важный вывод выделены

Переход к следующей мысли.
```

**Правило:** Один абзац = одна мысль. Максимум 3-4 строки.

## Техники: удержание внимания

### 1️⃣ Открытые петли (Open Loops)

Создаём вопрос → Мозг хочет ответ → Читает дальше.

**Почему это работает (механизм):**

```
Неопределённость → Мозг активирует режим поиска:

┌─────────────────────────────────────────────────────────┐
│  Открытая петля создаёт НЕОПРЕДЕЛЁННОСТЬ               │
│        │                                                │
│        ▼                                                │
│  Мозг не любит неопределённость                         │
│  (требует больше когнитивных ресурсов)                  │
│        │                                                │
│        ▼                                                │
│  Начинает "симулировать" возможные ответы              │
│  (часто негативные сценарии)                           │
│        │                                                │
│        ▼                                                │
│  Желание закрыть петлю = получить ответ                │
│        │                                                │
│        ▼                                                │
│  Продолжает читать, пока не получит closure            │
└─────────────────────────────────────────────────────────┘

Связь с эффектом Зейгарник: открытая петля = незавершённая
задача, которую мозг "держит активной".
```

**Примеры открытых петель:**

```
"Но есть одна проблема, о которой не говорят..."
                    ↓
Читатель: "Какая? Расскажи!"
                    ↓
Продолжает читать

"Через минуту я покажу, как это исправить. Но сначала..."
                    ↓
Создали ожидание
                    ↓
Читатель ждёт

"Это работало, пока не случилось X..."
                    ↓
Интрига
                    ↓
Хочется узнать что
```

**Антипримеры (когда открытые петли ВРЕДЯТ):**

```
❌ Слишком много петель одновременно:

"Я расскажу о трёх вещах. Первая удивит вас.
 Вторая изменит ваше понимание. А третья...
 но сначала предыстория..."

→ Читатель: "Сколько можно? Просто скажи уже!"
→ Раздражение вместо интереса


❌ Петля без реального payoff:

"Эта техника изменит вашу жизнь..."
[...500 слов воды...]
"...просто ставьте цели!"

→ Читатель: "Это и так знаю. Кликбейт."
→ Потеря доверия навсегда


❌ Петля в неподходящем контексте:

Документация API: "Вы не поверите, что делает
этот метод..."

→ Читатель: "Просто покажи сигнатуру!"
→ Неуместно для утилитарного контента
```

**Связь с когнитивными искажениями ([[cognitive-biases]]):**
- **Curiosity gap** — мозг стремится заполнить пробел в знаниях
- **FOMO (Fear of Missing Out)** — страх пропустить важное
- **Completion bias** — желание завершить начатое

**Важно:** Открытую петлю НУЖНО закрыть. Иначе — кликбейт и потеря доверия.

### 2️⃣ Паттерн-прерыватели

```
Читатель входит в режим скроллинга...

        ↓ скролл ↓

Обычный текст
Обычный текст
Обычный текст

┌─────────────────────────────────┐
│  ⚡ НЕОЖИДАННЫЙ БЛОК            │
│                                 │
│  Визуально другой элемент       │
│  ломает паттерн и возвращает    │
│  внимание                       │
└─────────────────────────────────┘

Читатель: "О, что это?"
        → Вернулся к чтению
```

**Примеры паттерн-прерывателей:**
- Боксы с выводами или важной информацией
- Графики и диаграммы
- Неожиданные вопросы к читателю
- Короткая история или пример
- Смена формата (список после текста, цитата)

### 3️⃣ Мост между разделами

```
❌ Без моста:

## Раздел 1
...контент...

## Раздел 2
...контент...

(Читатель между разделами: "Наверное, хватит")

✅ С мостом:

## Раздел 1
...контент...

Это объясняет "что". Но "как" — совсем другая история.
                ↓
## Раздел 2
...контент...

(Читатель: "О, сейчас будет 'как', интересно")
```

### 4️⃣ Риторические якоря

Вовлекают читателя в диалог:

```
Прямые обращения:
- "Знакомо?"
- "Было такое?"
- "Задумывались об этом?"

Провокации:
- "Звучит логично. Но это ловушка."
- "Всё, что ты знал об X — неправда."
- "Спойлер: это не работает."

Эмпатия:
- "Если ты как я..."
- "Проходил через это"
- "Понимаю, звучит сложно"
```

## Подводные камни и честная правда

### Проблема 1: Техники без ценности = манипуляция

```
Кликбейт-ловушка:

Яркий заголовок + Открытые петли + Идеальная структура
                        ↓
                Пустой контент
                        ↓
              Читатель обманут
                        ↓
         Потеря доверия навсегда
```

**Правило:** Техники усиливают хороший контент. Плохой контент они не спасут, а сделают хуже (явный кликбейт).

### Проблема 2: Культ "читабельности" vs глубина

```
Есть темы, где короткие абзацы не работают:

✅ Подходит для чанкинга:          ❌ Не подходит:
─────────────────────────         ───────────────────────
- Гайды и туториалы               - Философские рассуждения
- Новости и обзоры                - Сложные технические концепции
- Практические советы             - Нарративные истории
- Списки и чеклисты               - Академические работы
```

**Важно:** Не все тексты должны быть "СДВГ-friendly". Иногда глубина требует концентрации.

### Проблема 3: Оптимизация на метрики убивает качество

```
Что оптимизируем:                Чем жертвуем:
────────────────────             ────────────────────
Время на странице                Может быть = путаница
Дочитываемость                   Короткие тексты = поверхностность
Клики по заголовкам              Кликбейт = недоверие
Шейры                            Провокации = искажение
```

### Проблема 4: Это не универсально

```
Разные аудитории — разные ожидания:

Хабр-аудитория:     Ценит глубину, терпит длинные тексты
Twitter/X:          Максимум 280 символов, скорость
LinkedIn:           Профессионализм, "thought leadership"
Telegram-каналы:    Короткие форматы, регулярность
Документация:       Точность важнее увлекательности
```

### Проблема 5: СДВГ-friendly ≠ для людей с СДВГ

```
"СДВГ-friendly контент" — это маркетинговый термин.

Реальный СДВГ — это нейробиологическое состояние,
не просто "короткое внимание".

Люди с СДВГ могут:
✓ Часами читать интересную тему (гиперфокус)
✗ Не дочитать скучное, даже если оно "оптимизировано"

Ключевое: ИНТЕРЕС, не формат.
```

## Практический чеклист

### Перед написанием

```
□ Кто читатель? Что он уже знает?
□ Какую одну мысль должен унести?
□ Почему ему важно дочитать?
□ Какой формат подходит (гайд, размышление, новость)?
```

### Структура

```
□ TL;DR или главный вывод в начале
□ Каждый раздел начинается с вывода
□ Абзацы 2-4 строки максимум
□ Визуальное разнообразие (списки, боксы, схемы)
□ Мосты между разделами
```

### Удержание внимания

```
□ Сильный первый абзац (хук)
□ Открытые петли закрыты
□ Паттерн-прерыватели каждые 300-500 слов
□ Прямые обращения к читателю
□ Конкретика вместо абстракций (цифры, примеры)
```

### Честность

```
□ Заголовок соответствует содержанию
□ Обещания выполнены
□ Сложность темы отражена, не упрощена до кликбейта
□ Источники указаны
```

## Actionable: что делать

**Для следующего текста:**
- Напиши TL;DR первым
- Первый абзац = ответ на "зачем читать"
- Разбей на блоки по 2-4 строки
- Добавь хотя бы один паттерн-прерыватель

**Для практики:**
- Возьми свой старый текст → переформатируй по принципам
- Прочитай только заголовки и первые предложения → понятен ли смысл?
- Попроси кого-то прочитать → где остановился?

**Для системы:**
- Используй шаблоны с встроенной структурой
- Чеклист перед публикацией
- Анализируй, где читатели уходят (если есть аналитика)

## Проверь себя

> [!question]- Как эффект Зейгарник применяется в написании технических текстов для удержания внимания?
> Эффект Зейгарник: незавершённые задачи запоминаются лучше завершённых. В тексте это "открытые петли" -- вопросы или обещания в начале, ответ на которые даётся позже. Мозг "держит открытой вкладку" и мотивирован дочитать. Пример: "Через минуту я покажу технику, которая увеличила retention на 400%..." -- читатель не может остановиться, пока "петля" не закрыта.

> [!question]- 79% пользователей сканируют, а не читают. Как адаптировать техническую документацию под F-паттерн чтения?
> F-паттерн (Nielsen Norman Group, eye-tracking): читают первую строку полностью, затем сканируют начала строк, внимание падает вниз. Адаптация: (1) главный вывод -- в первом предложении (не в конце); (2) заголовки как самостоятельные утверждения; (3) ключевые слова -- в начале строк; (4) визуальная структура: списки, bold, code blocks; (5) короткие абзацы (3-4 строки). Текст должен быть понятен при сканировании, а не только при полном прочтении.

> [!question]- Почему время фокуса на экране упало с 2.5 минут (2004) до 47 секунд (2016) и как это влияет на написание технических текстов?
> Это не "люди стали тупее" -- это адаптация к среде с ~670+ единицами контента в день. Мозг научился быстро оценивать "стоит ли тратить время". Для авторов: (1) первые 10 секунд определяют, прочитают ли дальше; (2) TL;DR обязателен; (3) визуальная привлекательность (структура, пробелы) важнее "правильной" структуры; (4) каждый абзац должен оправдывать внимание читателя.

## Ключевые карточки

Что такое F-паттерн чтения?
?
Паттерн движения глаз при чтении веб-страниц (Nielsen Norman Group, eye-tracking). Читают первую строку, затем сканируют начала строк, внимание падает по F-образной форме. 79% сканируют, 16% читают полностью, ~5% дочитывают до конца.

Что такое эффект Зейгарник?
?
Незавершённые задачи запоминаются лучше завершённых (Блюма Зейгарник, 1927). Мозг "держит открытой вкладку" для незаконченного. В текстах используется как "открытые петли" -- вопросы и обещания, ответ на которые даётся позже, мотивируя дочитать.

Каково среднее время фокуса на одном экране?
?
47 секунд (2016, Gloria Mark, UC Irvine). Упало с 2.5 минут в 2004 году -- падение ~70% за 12 лет. Это адаптация к среде с ~670+ единицами контента в день, а не снижение интеллекта.

Почему главное должно быть в первом абзаце?
?
Из-за F-паттерна и 47-секундного внимания: читатель решает в первые 10 секунд, стоит ли продолжать. "Перевёрнутая пирамида" (из журналистики): вывод -> аргументы -> детали. Если вывод в конце -- 95% его не увидят.

Как "открытые петли" удерживают внимание?
?
Мозг стремится к завершённости (closure). Незакрытый вопрос или обещание создаёт "психологическое напряжение" (теория поля Левина), которое мотивирует продолжить чтение. Пример: "Одна техника увеличила engagement на 400%. Вот что это за техника..." -- читатель не может остановиться.
