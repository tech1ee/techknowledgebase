# Шаблон материала по коммуникации

> **Этот шаблон обязателен для КАЖДОГО материала в communication/.**
> Адаптирован из template-deep-dive.md для soft skills.

---

## Принципы

1. **IT-контекст** — все примеры из разработки: code review, sprint planning, 1-on-1
2. **Готовые скрипты** — минимум 5 фраз/templates для каждого use case
3. **Научное обоснование** — психология, исследования, статистика
4. **Практика > теория** — но теория объясняет ПОЧЕМУ практика работает

## Структура материала (12 секций)

### 1. TL;DR (2-3 предложения)

```markdown
> **TL;DR:** [Навык] позволяет [результат] через [механизм].
> Ключевая идея: [одно предложение].
> Применяется в: [2-3 IT-сценария].
```

```
ИССЛЕДОВАНИЕ:
□ /deep-researcher выполнен ПЕРЕД написанием
□ 20+ источников проанализированы
□ Научные исследования включены

СТРУКТУРА:
□ 12 обязательных секций присутствуют
□ 2000-4000 слов
□ Визуальный прерыватель каждые 200-300 слов

СОДЕРЖАНИЕ:
□ IT-контекст в примерах (code review, sprint, 1-on-1)
□ Минимум 5 скриптов/templates
□ Минимум 3 use cases с пошаговым разбором
□ "Когда НЕ использовать" раскрыто
□ Психологическое обоснование (исследования, статистика)

КАЧЕСТВО ТЕКСТА:
□ Нет AI-маркеров (delve, robust, crucial)
□ Нет канцеляризмов
□ Каждое слово на месте — нет воды
□ Конкретные цифры вместо "много/часто"

СВЯЗИ:
□ Prerequisites указаны как [[ссылки]]
□ Unlocks указаны
□ Связи с career/ section
```

## Проверь себя

> [!question]- Почему шаблон требует IT-аналогию для каждого термина в секции "Терминология"? Как это связано с теорией обучения?
> IT-аналогии активируют существующие нейронные связи читателя (transfer of learning). Разработчик уже понимает, что такое interface, dependency injection, middleware — и когда термин из психологии объясняется через знакомую IT-концепцию, новое знание "цепляется" к существующему. Это принцип elaborative encoding: чем больше связей у нового понятия, тем лучше оно запоминается.

> [!question]- В чеклисте качества указано "Нет AI-маркеров (delve, robust, crucial)". Почему это требование важно для образовательного материала по коммуникации?
> AI-маркеры сигнализируют, что текст написан без редактуры и авторского голоса. В материалах по коммуникации это особенно критично: если ты учишь людей писать конкретно и без воды, сам текст должен быть образцом. Слова типа "delve" и "robust" — это именно та расплывчатость, которую шаблон учит устранять ("Конкретные цифры вместо много/часто").

> [!question]- Почему шаблон требует минимум 5 скриптов и 3 use cases? Можно ли ограничиться теоретическим объяснением?
> Коммуникационные навыки — это процедурное знание (как ездить на велосипеде), а не декларативное (знать формулу). Без готовых скриптов человек понимает теорию, но в стрессовой ситуации (code review, конфликт) откатывается к привычным паттернам. 5 скриптов дают вариативность, 3 use cases — контекст для применения. Это принцип [[deliberate-practice]]: нужны конкретные действия, а не абстракции.

> [!question]- Сравни секцию "Когда использовать / НЕ использовать" в этом шаблоне с аналогичной секцией в [[template-leadership]]. В чём принципиальная разница?
> В communication-шаблоне фокус на ситуациях и альтернативах (когда лучше использовать другой навык). В leadership-шаблоне эта секция называется "Trade-offs" и включает таблицу "За/Против" — потому что лидерские решения всегда имеют компромиссы. Коммуникационный навык можно заменить другим, а лидерский подход — нет, у него есть цена.

## Ключевые карточки

Сколько обязательных секций в шаблоне материала по коммуникации?
?
12 секций: TL;DR, Зачем это нужно, Для кого, Терминология, Как это работает, Пошаговый процесс, Скрипты и Templates, Распространённые ошибки, Когда использовать/НЕ использовать, Практические задачи, Связанные темы, Источники.

Какие 4 принципа заложены в основу шаблона?
?
1) IT-контекст — примеры из code review, sprint planning, 1-on-1. 2) Готовые скрипты — минимум 5 фраз/templates. 3) Научное обоснование — психология и исследования. 4) Практика > теория — но теория объясняет ПОЧЕМУ.

Какой минимальный объём материала по шаблону?
?
2000-4000 слов, с визуальным прерывателем каждые 200-300 слов. Минимум 5 скриптов, 3 use cases с пошаговым разбором.

Что должно быть в метаданных помимо стандартных полей (title, created, type)?
?
prerequisites (что изучить ДО), teaches (какие навыки даёт), unlocks (что открывает ПОСЛЕ). Эти поля создают граф зависимостей между материалами.

Из каких шагов состоит workflow создания материала?
?
8 шагов: deep-researcher → выписать модели/фреймворки → придумать IT-аналогии → написать 5+ скриптов → создать ASCII-визуализации → написать по шаблону → проверить по чеклисту → обновить MOC и связи.

Почему секция "Распространённые ошибки" требует поле "Почему это неправильно"?
?
Потому что одного примера ошибки недостаточно — нужно объяснить психологический механизм (почему так говорить вредно). Без понимания причины человек будет повторять ошибку в новых контекстах.

На основе какого шаблона создан template-communication?
?
На основе template-deep-dive.md, адаптированного для специфики soft skills. Ключевое отличие — акцент на готовых скриптах и IT-контексте в примерах.
