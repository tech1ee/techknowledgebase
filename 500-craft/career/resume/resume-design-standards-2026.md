# Resume Design Standards 2026: Pixel-Perfect Guidelines

## Executive Summary

One-page resume with every millimeter planned. ATS-friendly but visually polished.
Key principle: **Minimalism + Precision = Professional Impact**

---

## Page Setup

### Dimensions
| Market | Size | Dimensions |
|--------|------|------------|
| US/Canada | US Letter | 8.5 x 11 inches (215.9 x 279.4 mm) |
| Europe/International | A4 | 210 x 297 mm (8.27 x 11.69 inches) |

### Margins (Critical)
```
Standard: 1 inch (25.4mm) all sides
Minimum: 0.5 inch (12.7mm) - only when absolutely necessary
Sweet spot: 0.6-0.75 inch (15-19mm) - professional balance

NEVER go below 0.5 inch - causes ATS parsing issues
```

## Typography

### Font Selection (ATS-Safe Priority)
```
TIER 1 - Best ATS Compatibility:
1. Calibri (modern corporate standard)
2. Arial (universal, clean)
3. Helvetica (design-forward, professional)

TIER 2 - Good Options:
4. Garamond (elegant serif)
5. Georgia (screen-optimized serif)
6. Cambria (modern serif)

AVOID:
- Script fonts
- Decorative fonts
- Uncommon fonts not on all systems
```

### Font Sizes (Visual Hierarchy)
```
NAME:           18-24pt (22pt optimal)
SECTION TITLES: 11-14pt (12pt optimal)
JOB TITLES:     10-11pt (bold)
BODY TEXT:      10-11pt (11pt optimal)
CONTACT INFO:   9-10pt

NEVER below 10pt for any important text
```

### Font Weights
```
Name:            700 (Bold)
Section Headers: 600 (Semi-Bold)
Job Titles:      600 (Semi-Bold)
Company Names:   500-600
Body Text:       400 (Regular)
Dates:           400 (Regular, possibly italic)
```

### Line Height (Leading)
```
Body text: 1.15-1.35 (120-135%)
Tight: 1.0-1.15 (for dense content)
Relaxed: 1.4-1.5 (more white space)

Between sections: 1.5-2.0 line heights
Between bullets: 1.0-1.15
```

### Letter Spacing (Tracking)
```
Name (uppercase): -0.02em to -0.03em (slightly tighter)
Section headers: 0.05em to 0.1em (slightly wider)
Body text: 0 (default)
```

## Layout & Structure

### Single Column Rule
```
ATS systems struggle with:
- Multiple columns
- Tables
- Text boxes
- Graphics/images
- Headers/footers (content might be ignored)

ALWAYS use single-column, top-to-bottom flow
```

### Section Order (Skills-First for Tech)
```
1. HEADER (Name, Headline, Contact)
2. PROFESSIONAL SUMMARY (2-3 lines)
3. TECHNICAL EXPERTISE (skills first - 2026 trend)
4. PROFESSIONAL EXPERIENCE
5. EDUCATION (optional for seniors)
6. CERTIFICATIONS (if relevant)
```

### Visual Separators
```
Section dividers: Thin line (0.5-1pt) or extra spacing
Bullet style: Simple round (•) or dash (–)

AVOID:
- Arrows
- Checkboxes
- Custom icons
- Multiple colors
```

### Alignment
```
Header (name, contact): Centered
Everything else: Left-aligned

AVOID justified text - creates uneven word spacing
```

## Spacing Specifications

### Section Spacing
```
Before section title: 14-18pt
After section line: 6-8pt
Between sections: 16-20pt
```

### Experience Items
```
Between jobs: 10-14pt
Title to bullets: 4-6pt
Between bullets: 2-4pt
Bullet indent: 12-16pt (from left margin)
```

### Header Spacing
```
Name to headline: 2-4pt
Headline to contact: 6-8pt
Contact items gap: 6-8pt (between separators)
Header to first section: 12-16pt
```

### Skills Section
```
Category line spacing: 3-5pt
Inline skill separator: pipe (|) with 4-6pt space
```

## Color Guidelines

### Professional Palette
```
Primary text:     #000000 (pure black) or #1a1a1a (soft black)
Secondary text:   #4a4a4a to #666666 (dark gray)
Links/accents:    #0066cc (professional blue) or #2b5797
Section lines:    #cccccc to #e0e0e0 (light gray)

AVOID:
- Bright colors
- Multiple accent colors
- Background colors
```

### Contrast Requirements
```
Minimum contrast ratio: 4.5:1 (WCAG AA)
Text on white background: use #333333 or darker
```

## Content Density

### One-Page Rule
```
FOR SENIOR (5+ years):
- One page strongly preferred
- Two pages acceptable only if truly necessary
- Every bullet must earn its place

DENSITY BALANCE:
- Too sparse = lacks substance
- Too dense = hard to read
- Target: 60-70% text, 30-40% white space
```

### Bullet Points
```
Per job:          3-5 bullets (max 5)
Bullet length:    1-2 lines (max 150 characters)
Format:           Action verb + Task + Result (metric)
```

## PDF Export Specifications

### File Properties
```
Format: PDF (not Word for submission)
PDF version: 1.4 or higher
Fonts: Embedded
Searchable: Text must be selectable/copyable
File name: FirstName_LastName_Resume.pdf
File size: Under 1MB
```

### Quality Check
```
[ ] Text is selectable (not image)
[ ] Links are clickable
[ ] No artifacts or overflow
[ ] Consistent spacing
[ ] Clean page break (if 2 pages)
```

## Common Mistakes to Avoid

### Design Mistakes
```
- Using tables for layout
- Multiple columns
- Icons or graphics
- Photo on resume (US/UK)
- Inconsistent spacing
- Too many font sizes/weights
- Centered body text
- Justified text
```

### Content Mistakes
```
- No metrics in bullets
- Passive voice ("Responsible for")
- Buzzwords without evidence
- Walls of text (no white space)
- Outdated information
```

## Проверь себя

> [!question]- Почему margins ниже 0.5 inch вызывают проблемы с ATS, и какой оптимальный диапазон?
> ATS парсеры сканируют документ слева направо, и слишком узкие margins могут обрезать текст или нарушить parsing — символы попадают в "мёртвую зону". Оптимальный sweet spot: 0.6-0.75 inch (15-19mm) — профессиональный баланс между плотностью контента и читаемостью. Standard 1 inch — безопасно, но теряется пространство. Minimum 0.5 inch — только если абсолютно необходимо вместить контент.

> [!question]- Дизайнер предлагает двухколоночный layout с иконками для резюме. Назови 3 причины отказаться.
> (1) ATS не парсит columns правильно — текст смешивается между колонками, keywords теряются. (2) Иконки/графика не читаются ATS — парсер видит пустое место вместо информации. (3) Tables и text boxes ломают text flow — ATS ожидает top-to-bottom single column. ВСЕГДА: single-column, top-to-bottom flow. Даже "красивые" шаблоны из Canva/Figma проваливают ATS test.

> [!question]- Какой font size для body text оптимален и почему нельзя опускаться ниже 10pt?
> Оптимальный body text: 10-11pt (11pt optimal). Ниже 10pt: (1) трудно читать при печати и скане, (2) некоторые ATS масштабируют PDF и мелкий текст становится нечитаемым, (3) рекрутер с 6 секундами на скан физически не прочитает мелкий шрифт. Visual hierarchy: Name 22pt -> Section titles 12pt -> Job titles 10-11pt bold -> Body 10-11pt -> Contact 9-10pt.

> [!question]- Резюме готово в PDF. Какие 5 пунктов quality check нужно пройти перед отправкой?
> (1) Text is selectable — не image-based PDF (ATS не может парсить картинки). (2) Links are clickable — LinkedIn, GitHub, email. (3) No artifacts or overflow — текст не обрезается. (4) Consistent spacing — одинаковые отступы между sections, bullets, jobs. (5) File size under 1MB, filename: FirstName_LastName_Resume.pdf. Дополнительно: fonts embedded, PDF version 1.4+.

## Ключевые карточки

Margins — оптимальные значения?
?
Sweet spot: 0.6-0.75 inch (15-19mm). Standard: 1 inch. Minimum: 0.5 inch (только если необходимо). НИКОГДА ниже 0.5 inch — ATS parsing issues.

Fonts — Tier 1 для ATS?
?
(1) Calibri (modern corporate standard), (2) Arial (universal, clean), (3) Helvetica (design-forward). Avoid: script fonts, decorative, uncommon. Font sizes: Name 18-24pt, sections 11-14pt, body 10-11pt.

Single Column Rule — почему?
?
ATS не парсит: multiple columns, tables, text boxes, graphics/images, headers/footers. ВСЕГДА single-column, top-to-bottom flow. Это не рекомендация — это требование для 97% компаний с ATS.

Visual hierarchy — font weights?
?
Name: 700 (Bold). Section headers: 600 (Semi-Bold). Job titles: 600. Company names: 500-600. Body text: 400 (Regular). Dates: 400 (possibly italic). Consistent weights = professional appearance.

Content density — правило баланса?
?
Target: 60-70% text, 30-40% white space. Too sparse = lacks substance. Too dense = hard to read. Per job: 3-5 bullets max. Bullet length: 1-2 lines (max 150 chars). One page strongly preferred для Senior.

Color guidelines — professional palette?
?
Primary text: #000000 или #1a1a1a. Secondary: #4a4a4a to #666666. Links/accents: #0066cc. Section lines: #cccccc to #e0e0e0. Avoid bright colors, multiple accents, backgrounds. Contrast ratio min 4.5:1.

PDF export checklist?
?
Format: PDF (not Word). Fonts: embedded. Text: selectable. Links: clickable. File name: FirstName_LastName_Resume.pdf. Size: under 1MB. PDF version 1.4+.

Section order для Tech (Skills-First)?
?
(1) Header (Name, Headline, Contact), (2) Professional Summary (2-3 lines), (3) Technical Expertise (skills first — 2026 trend), (4) Professional Experience, (5) Education, (6) Certifications.
