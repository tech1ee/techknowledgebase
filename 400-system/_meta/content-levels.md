# Уровни качества контента

Контент делится на 5 уровней (0-4). Уровни 0-1 — информация, которую можно нагуглить. Уровни 2-4 — уникальный контент с экспертизой.

**Научная база:** Bloom's Taxonomy (1956, revised 2001) + Vygotsky ZPD (1978).

---

## Пирамида уровней

### Уровень 0: Курирование
"Вот ссылка, годная статья!"

Даже просто поделиться хорошим материалом — уже польза. Умело фильтровать входящий поток информации — недооценённый навык.

**Вариант 0-ru:** Перевод. Примитивный способ добавить ценности, но помним — читать переводы вместо оригиналов вредно для развития.

### Уровень 1: Суммаризация с инсайтами
"Вот основные тезисы статьи"

Найти главные, а лучше — неочевидные инсайты. Критерий: твои инсайты должны быть сильнее, чем LLM-суммаризация. Иначе — это уровень 0.

### Уровень 2: Кросс-тематические связи
Добавить экспертизу из смежных областей.

Для статьи о будущем AI это могут быть:
- Данные о hype cycles
- Наблюдения из бизнеса
- Исторические примеры

Тут начинается уникальность — это уже нельзя нагуглить.

### Уровень 3: Actionable инсайты
Самое важное: любой читатель их ищет (осознанно или нет).

Описать способ, как читатель может применить информацию для своей выгоды. Это win-win: полезно читателю, ценно как контент.

**Эффект:** тактический.

### Уровень 4: Возможности
Знание, которого не было у читателя и которое может изменить его жизнь:
- Стартап-акселератор
- Программа магистратуры
- Идея для стартапа
- Карьерный путь

**Эффект:** стратегический.

## Экспертиза автора и формат

### Начинающий в теме
Формат: "Учусь сам и рассказываю, что узнал"

Не стесняться начального уровня — это:
- Привлекает единомышленников на том же пути
- Защищает от "синдрома эксперта" без реального опыта

### Эксперт в теме
Два пути:

**А) Писать самое важное из опыта**
- Сложное, неоткликающееся у обывателя
- Аудитория: 27 человек на том же пути
- Лучше организовать закрытое сообщество

**Б) Писать для широкого круга**
- Истории (быстро закончатся)
- Наблюдения (генерируют хейтеров)
- Теория (в большинстве областей её 1%, остальное — практика)

Итог: контент сводится к "как стать X" (предпринимателем, ML-инженером, и т.д.)

## Ключевые инсайты

### Драйвер любого блога с аудиторией >100 человек
Лейтмотив "как сделать жизнь лучше":
- Как стать ML инженером
- Как начать бизнес
- Как устроиться в FAANG

### Парадокс ценного контента
Самое ценное — одновременно самое контринтуитивное и плохо принимаемое широкой аудиторией.

### Выход для экспертов
Писать не про узкие темы, а про их объединяющий фундаментальный уровень:
- Не про стартапы, номадизм, дизайн
- А про цели: свободу, востребованность, безопасность, счастье

Выражать в прикладной форме через призму своего опыта.

## Применение к Knowledge Vault

При создании заметок спрашивать себя:
1. На каком уровне этот контент?
2. Могу ли добавить кросс-тематические связи? (→ уровень 2)
3. Есть ли actionable инсайт? (→ уровень 3)
4. Открывает ли это возможности для читателя? (→ уровень 4)
5. Текст написан как глава учебника? (80%+ пояснительный текст, код — иллюстрация)
6. Указаны признанные источники? (книги, papers)

**Цель:** минимум уровень 2 для каждой заметки.

## Bloom's Taxonomy: когнитивные уровни

> **Источник:** Bloom (1956), Anderson & Krathwohl (revised, 2001)

```
Уровень 6: Create     ← Создавать новое
Уровень 5: Evaluate   ← Оценивать, критиковать
Уровень 4: Analyze    ← Разбирать на части, находить связи
Уровень 3: Apply      ← Применять знания на практике
Уровень 2: Understand ← Объяснять своими словами
Уровень 1: Remember   ← Запоминать факты
```

### Связь с типами контента

| Тип контента | Bloom Level | Цель читателя |
|--------------|-------------|---------------|
| REFERENCE | 1-2: Remember/Understand | Быстро найти синтаксис |
| EXPLAINER | 2: Understand | Объяснить концепцию своими словами |
| TUTORIAL | 3: Apply | Применить знания на практике |
| DEEP DIVE | 4: Analyze | Понять структуру и связи |
| COMPARISON | 5: Evaluate | Критически оценить варианты |

### Связь с уровнями контента

| Уровень контента | Bloom Level | Пример |
|------------------|-------------|--------|
| 0: Курирование | 1: Remember | "Вот ссылка" |
| 1: Суммаризация | 2: Understand | "Вот ключевые тезисы" |
| 2: Кросс-связи | 4: Analyze | "Как это связано с X" |
| 3: Actionable | 3: Apply | "Как применить для Y" |
| 4: Возможности | 5-6: Evaluate/Create | "Это открывает Z" |

## Зона ближайшего развития (ZPD)

> **Источник:** Vygotsky (1978)

Эффективное обучение происходит в зоне между "могу сам" и "не могу даже с помощью".

```
┌─────────────────────────────────────────┐
│ Зона актуального развития               │
│ (уже умеет — не нужно объяснять)        │
│    ┌─────────────────────────────┐      │
│    │ Зона ближайшего развития    │      │
│    │ (может с помощью) ← ЦЕЛЬ    │      │
│    │    ┌───────────────────┐    │      │
│    │    │ Пока недоступно   │    │      │
│    │    │ (нужны prereqs)   │    │      │
│    │    └───────────────────┘    │      │
│    └─────────────────────────────┘      │
└─────────────────────────────────────────┘
```

### Применение к контенту

| Проверка | Что делать |
|----------|------------|
| Материал слишком простой | Добавить глубину или убрать |
| Материал слишком сложный | Добавить prerequisites |
| Материал в ZPD | Идеально — сохраняем |

**Практика:**
- Указывать prerequisites в начале
- Использовать аналогии с известным (scaffolding)
- Строить от простого к сложному
- Каждый файл intermediate+ понятен с нуля (при наличии prerequisites)
- Минимальные требования к глубине: [[depth-standard]]

## Чеклист перед публикацией

> 5 пунктов — оптимум для чеклиста (Gawande, 2009)

```
□ Уровень контента >= 2 (есть своя экспертиза)
□ Bloom level соответствует типу контента
□ Материал в ZPD целевой аудитории
□ Prerequisites указаны (если нужны)
□ Есть практическое применение (actionable)
```
