# Когнитивная наука: правила создания контента

> Каждое правило здесь основано на исследованиях. Без ссылки на источник — не правило.

---

## Зачем этот файл

Текущие правила содержат противоречия с когнитивной наукой:
- 5 вопросов в Lead Paragraph превышают рабочую память (4±1)
- 8 вариантов открытий создают cognitive overload при выборе
- Чеклист из 19 пунктов неэффективен (>7 снижает compliance)

Этот файл — научная база для всех остальных правил.

---

## Фундаментальные ограничения

### Рабочая память: 4±1 элемент

**Источник:** Cowan, 2000; Miller, 1956 (уточнён)

Старое правило "7±2" (Miller) пересмотрено. Реальный лимит — 4 элемента, максимум 5 при группировке.

```
Применение:
- Вводный абзац: максимум 3-4 ключевых элемента
- Варианты выбора: максимум 4 опции
- Чеклисты: 5-7 пунктов на один чеклист
- Новых концепций в разделе: максимум 4
```

**Механизм:** Рабочая память — "бутылочное горлышко" обучения. Превышение лимита → часть информации теряется до попадания в долговременную память.

### Attention Span: 47 секунд

**Источник:** Gloria Mark, UC Irvine, 2016

Среднее время фокуса на одном экране сократилось с 2.5 минут (2004) до 47 секунд (2016).

```
Применение:
- Визуальный прерыватель каждые 200-300 слов
- Ключевая мысль в первых 2-3 предложениях
- Структура "сканируемая": заголовки, списки, таблицы
```

### Cognitive Load Theory

**Источник:** Sweller, 1988

Три типа когнитивной нагрузки:
1. **Intrinsic** — сложность самого материала (неизбежна)
2. **Extraneous** — плохая организация материала (устраняема)
3. **Germane** — усилия на формирование схем (полезная)

```
Применение:
- Убирать extraneous load: чёткая структура, без лишнего
- Не снижать germane load: сохранять "полезные трудности"
- Управлять intrinsic: разбивать сложное на чанки
```

## Принципы эффективного обучения

### Spacing Effect (интервальное повторение)

**Источник:** Ebbinghaus, 1885; Bjork, 1994

Распределённое обучение эффективнее массированного. Retention через неделю: 80-90% vs 30%.

```
Применение в тексте:
- Ключевая идея появляется минимум 2 раза в разных контекстах
- Во введении: простое объяснение
- В примерах: практическое применение
- В связях: как соотносится с другими концепциями
```

### Desirable Difficulties (полезные трудности)

**Источник:** Robert Bjork, Elizabeth Bjork, UCLA, 1994

Трудности, которые замедляют обучение сейчас, но улучшают долгосрочное запоминание:
- **Testing Effect:** тестирование лучше перечитывания
- **Generation Effect:** генерация лучше узнавания
- **Interleaving:** чередование лучше блоков

```
Применение:
- Вопросы ДО объяснения (pre-questions), не после
- Пробелы для самостоятельного вывода
- Контраст ❌/✅ для активного сравнения
- Задачи на применение, не на узнавание
```

**Warning:** Desirable difficulties работают только при наличии базовых знаний. Для новичков — сначала foundation.

### Testing Effect (эффект тестирования)

**Источник:** Roediger & Karpicke, 2006

Попытка вспомнить информацию укрепляет память эффективнее, чем повторное чтение. Эффект: 2-3x улучшение retention.

```
Применение:
- "Проверь себя" секция (скрытый ответ)
- Вопросы перед объяснением
- Контраст для анализа (что правильно/неправильно?)
- Задачи на применение
```

### Chunking (группировка)

**Источник:** Miller, 1956; Gobet et al., 2001

Группировка информации в значимые единицы увеличивает effective capacity рабочей памяти.

```
Применение:
Максимум 4 новых концепции на раздел.

Каждый чанк:
1. Название (создаёт метку в памяти)
2. Определение (1-2 предложения)
3. Пример (конкретный)
4. Связь с известным (аналогия)
```

### Dual Coding Theory

**Источник:** Paivio, 1986

Информация, представленная визуально И вербально, запоминается лучше.

```
Применение:
- Схемы/диаграммы + текстовое объяснение
- Код + комментарии
- Таблица + пояснение
```

**Важно:** Визуал не заменяет текст, а дополняет. Схема без объяснения = cognitive load без пользы.

**Приоритет:** Текст — основной носитель знания (80%+). Визуал (диаграммы, таблицы) и код — дополнения к тексту. Никогда наоборот: не строить материал вокруг кода с краткими пояснениями. Код иллюстрирует теорию, а не заменяет её.

### Scaffolding (выстраивание опоры)

**Источник:** Bruner, 1976; Wood, Bruner & Ross, 1976

Обучение идёт от простого к сложному. Каждый новый элемент опирается на уже понятое. Убери одну ступень — вся конструкция рушится. Scaffolding — это временные опоры (аналогии, упрощения, предварительные объяснения), которые помогают добраться до сложной идеи.

```
Применение:
- Каждый раздел начинается с того, что читатель уже знает
- Новая концепция вводится через аналогию с известным
- Prerequisites указаны явно — не предполагать знания
- Порядок разделов = порядок обучения (не алфавитный)
- Каждый термин объяснён при первом появлении
```

**Warning:** Scaffolding не означает "упрощай всё". Это означает "строй мост от известного к неизвестному". Сложность должна нарастать, но каждый шаг — посильный.

### Source Authority (авторитет источника)

**Источник:** Petty & Cacioppo, Elaboration Likelihood Model, 1986

Материал с указанием признанных источников воспринимается как более авторитетный и запоминается лучше. Это не только про доверие — ссылки дают читателю путь для углубления.

```
Применение:
- Каждый файл intermediate+: 2+ признанных источника
- Источники = книги и papers (приоритет), не блоги
- Формат: Автор (год) — зачем читать
- Источники в конце файла + inline-ссылки в тексте при первом упоминании
```

**Механизм:** Peripheral route to persuasion — даже если читатель не проверяет источники, их наличие повышает perceived credibility и мотивацию к обучению.

## Принципы эффективных чеклистов

**Источник:** Atul Gawande, "The Checklist Manifesto", 2009

Чеклисты из 7+ пунктов менее эффективны. Оптимум: 5-7 пунктов.

```
Применение:
БЫЛО: 1 чеклист из 19 пунктов
СТАЛО: 3 микро-чеклиста по 5-7 пунктов

- Чеклист структуры (5 пунктов)
- Чеклист содержания (6 пунктов)
- Чеклист когнитивной эффективности (5 пунктов)
```

**Механизм:** Длинные чеклисты воспринимаются как "wall of text" → пропуск пунктов → ложное чувство выполнения.

## Принципы выбора

### Paradox of Choice

**Источник:** Barry Schwartz, 2004; Iyengar & Lepper, 2000

Слишком много вариантов → паралич выбора. 24 варианта джема → 3% покупок. 6 вариантов → 30% покупок.

```
Применение:
- Варианты открытий: максимум 4 типа (было 8)
- Дерево решений вместо списка опций
- Рекомендация по умолчанию: "если не знаешь — выбирай X"
```

## Зона ближайшего развития (ZPD)

**Источник:** Vygotsky, 1978

Эффективное обучение происходит в зоне между "могу сам" и "не могу даже с помощью".

```
┌─────────────────────────────────────────┐
│ Зона актуального развития               │
│ (уже умеет)                             │
│    ┌─────────────────────────────┐      │
│    │ Зона ближайшего развития    │      │
│    │ (может с помощью) ← ЦЕЛЬ    │      │
│    │    ┌───────────────────┐    │      │
│    │    │ Пока недоступно   │    │      │
│    │    │ (слишком сложно)  │    │      │
│    │    └───────────────────┘    │      │
│    └─────────────────────────────┘      │
└─────────────────────────────────────────┘
```

```
Применение:
- Контент для разных уровней (Junior/Middle/Senior)
- Prerequisites чётко указаны
- Материал строится на известном (мост к новому)
```

## Bloom's Taxonomy: уровни когнитивных целей

**Источник:** Bloom, 1956; Anderson & Krathwohl (revised), 2001

```
Уровень 6: Create     ← Создавать новое
Уровень 5: Evaluate   ← Оценивать, критиковать
Уровень 4: Analyze    ← Разбирать на части
Уровень 3: Apply      ← Применять знания
Уровень 2: Understand ← Объяснять своими словами
Уровень 1: Remember   ← Запоминать факты
```

```
Применение к типам контента:

EXPLAINER → Understand (уровень 2)
  Цель: читатель может объяснить концепцию своими словами

TUTORIAL → Apply (уровень 3)
  Цель: читатель может применить знания на практике

DEEP DIVE → Analyze (уровень 4)
  Цель: читатель понимает структуру и связи

COMPARISON → Evaluate (уровень 5)
  Цель: читатель может критически оценить варианты
```

## Чеклист когнитивной эффективности

Проверять для каждого материала:

```
РАБОЧАЯ ПАМЯТЬ:
□ Вводный абзац: максимум 3-4 элемента
□ Вариантов выбора: максимум 4
□ Новых концепций в разделе: максимум 4

ВНИМАНИЕ:
□ Визуальный прерыватель каждые 200-300 слов
□ Ключевая мысль в первых 2-3 предложениях
□ Структура "сканируемая" (заголовки, списки)

ОБУЧЕНИЕ:
□ Ключевая идея повторяется в разных контекстах (spacing)
□ Есть элементы активного вовлечения (вопросы, задачи)
□ Визуал + текст вместе (dual coding)
□ Материал строится на известном (scaffolding)

ПРИМЕНИМОСТЬ:
□ Когнитивный уровень соответствует типу контента
□ Prerequisites указаны
□ Есть раздел "когда НЕ применять" (boundaries)
```
